Data Storage

Fundamentals of data storage: from cloud infrastructure to relational databases like MySQL. Learn how information is organized, stored, and accessed efficiently.

1. Introduction to Data

In the digital era, data is one of the most valuable assets for individuals, companies, and governments. Understanding how data is generated, stored, processed, and secured is essential for web developers and IT professionals. Additionally, modern infrastructure heavily relies on cloud computing to provide scalable, reliable, and cost-efficient solutions for managing data at scale.

1.1 What is Data?

Data refers to pieces of information collected for reference or analysis. It can take various forms:

  • Structured data: organized in tables (e.g., databases).
  • Unstructured data: images, videos, audio, social media posts, etc.
  • Semi-structured data: emails, XML/JSON files, log files.

Data can be collected through sensors, web forms, APIs, or user activity. To use it efficiently, it needs to be stored in a format that allows easy access, processing, and analysis.

2. Cloud Computing Overview

Cloud computing refers to the delivery of computing services—such as servers, storage, databases, networking, software, analytics, and intelligence—over the internet ("the cloud") to offer faster innovation, flexible resources, and economies of scale. Instead of owning and maintaining physical servers or infrastructure, individuals and organizations can access computing resources on demand via a cloud provider.

2.1 Key Characteristics of Cloud Computing

Cloud computing has transformed how digital services are delivered, shifting from locally hosted infrastructures to flexible, on-demand resources accessed over the internet. To fully understand this paradigm, it's important to explore its fundamental characteristics. In this section, we will examine the key attributes that define cloud computing—such as scalability, elasticity, measured service, and resource pooling—and how these characteristics make the cloud an ideal platform for modern applications and businesses.

  • On-Demand Self-Service: Users can provision resources like servers or storage automatically, without human intervention.
  • Broad Network Access: Services are available via standard interfaces (e.g., browsers, APIs).
  • Resource Pooling: Providers share resources among multiple customers using a multi-tenant model.
  • Rapid Elasticity: Resources can scale dynamically as needed, often appearing unlimited to users.
  • Measured Service: Usage is monitored and billed based on actual consumption.

2.2 Main Cloud Service Models

Cloud computing offers different service models to accommodate a variety of needs, from infrastructure provisioning to full application deployment. Understanding these models is essential for selecting the right type of cloud service depending on the project requirements. This section introduces the three primary cloud service models—Infrastructure as a Service (IaaS), Platform as a Service (PaaS), and Software as a Service (SaaS)—and highlights the key features, advantages, and use cases of each.

  • IaaS (Infrastructure as a Service): Offers fundamental resources like virtual machines and storage. Customers manage OS and applications.
    Examples: AWS EC2, Google Compute Engine, Azure Virtual Machines.
  • PaaS (Platform as a Service): Provides a ready-to-use platform for development and deployment without infrastructure management.
    Examples: Heroku, Google App Engine, Azure App Services.
  • SaaS (Software as a Service): Delivers software applications over the internet.
    Examples: Google Workspace, Microsoft 365, Dropbox.

2.3 Deployment Models

Cloud deployment models define how cloud services are made available to users and how the infrastructure is owned and managed. Choosing the right deployment model is crucial for meeting security, scalability, and compliance requirements. In this section, we explore the four main cloud deployment models—Public Cloud, Private Cloud, Hybrid Cloud, and Community Cloud—highlighting their characteristics, advantages, and common use cases.

  • Public Cloud: Services provided over the internet and shared across customers (e.g., AWS, Azure).
  • Private Cloud: Dedicated infrastructure for one organization, managed internally or by a third party.
  • Hybrid Cloud: Combines public and private clouds to allow data and apps to move between them.
  • Multi-Cloud: Use of services from multiple cloud providers to avoid vendor lock-in and improve flexibility.

2.4 Advantages of Cloud Computing

Cloud computing offers numerous advantages that have transformed how individuals, businesses, and governments store data, run applications, and manage IT infrastructure. From cost savings and scalability to increased flexibility and accessibility, the benefits of cloud services are significant across all sectors. In this section, we examine the key advantages of adopting cloud computing and how they contribute to greater efficiency and innovation.

  • Cost Efficiency: Pay only for what you use, no upfront hardware investment.
  • Scalability: Scale resources up or down quickly and automatically.
  • Performance: Access high-performance infrastructure across global regions.
  • Security: Enterprise-level security, data encryption, and compliance support.
  • Disaster Recovery: Built-in tools to ensure continuity and backups.

2.5 Common Use Cases

Cloud computing has become a foundational technology for a wide range of real-world applications. Its flexibility and power make it ideal for use cases across industries—from hosting websites and storing large volumes of data, to enabling remote collaboration and supporting artificial intelligence. In this section, we explore the most common and impactful ways organizations and individuals leverage the cloud in their daily operations.

  • Web hosting and web app deployment
  • Cloud storage and backups
  • Big data analytics and machine learning
  • IoT (Internet of Things) platforms
  • Streaming media services
  • Collaboration and productivity apps

In summary: Cloud computing has revolutionized the way we build, deploy, and scale digital solutions. It allows individuals and organizations to focus on innovation, efficiency, and global accessibility, while cloud providers manage the heavy infrastructure.

3. Databases Overview

A database is an organized collection of data that can be easily accessed, managed, and updated. Databases are essential in web development, powering features like user authentication, content management, and e-commerce transactions.

3.1 Types of Databases

  • Relational Databases (SQL): Use tables and enforce relationships using keys (e.g., MySQL, PostgreSQL).
  • NoSQL Databases: Designed for flexibility and scalability, often used with unstructured data (e.g., MongoDB, Firebase).
  • Cloud Databases: Hosted by cloud providers, offering scalability, backup, and high availability (e.g., Amazon RDS, Google Cloud SQL).

3.2 What is SQL?

SQL (Structured Query Language) is the standard language used to interact with relational databases. It allows you to define, manipulate, secure, and manage data through different categories of statements.

3.3 SQL Command Categories

To work effectively with relational databases, it’s important to understand the different types of operations you can perform using SQL. These operations are grouped into distinct command categories, each serving a specific purpose in how data is structured, accessed, modified, or secured. Before diving into individual statements, this section introduces the major categories of SQL commands and how they fit into the overall workflow of database management.

  • DDL (Data Definition Language)
    Used to define and modify the structure of database objects like tables, schemas, and indexes.
    • CREATE – Creates new tables or databases
    • ALTER – Modifies existing tables (add/remove columns, etc.)
    • DROP – Deletes tables or databases
    • TRUNCATE – Removes all rows from a table without logging individual deletions
  • DML (Data Manipulation Language)
    Deals with the actual data within the tables.
    • SELECT – Retrieves data from one or more tables
    • INSERT – Adds new rows to a table
    • UPDATE – Modifies existing data
    • DELETE – Removes specific rows
  • DCL (Data Control Language)
    Manages permissions and access control.
    • GRANT – Gives user access privileges
    • REVOKE – Removes access privileges
  • TCL (Transaction Control Language)
    Handles transactions to ensure data integrity and consistency.
    • COMMIT – Saves all changes made during the current transaction
    • ROLLBACK – Cancels all changes made in the current transaction
    • SAVEPOINT – Sets a point within a transaction to roll back to

3.4 Sample SQL Queries

Once you understand the structure of a database and the categories of SQL commands, it's time to see them in action. This section presents a series of common SQL queries that illustrate how to retrieve, insert, update, and delete data. These examples are designed to reinforce the concepts you've learned and provide practical templates you can adapt to your own projects. 

-- DDL: Create a table
CREATE TABLE users (
  id INT AUTO_INCREMENT PRIMARY KEY,
  name VARCHAR(100) NOT NULL,
  email VARCHAR(100) UNIQUE NOT NULL
);

-- DML: Insert a record
INSERT INTO users (name, email) VALUES ('Alice Smith', 'alice@example.com');

-- DML: Update a record
UPDATE users SET email = 'alice_new@example.com' WHERE id = 1;

-- DML: Select data
SELECT * FROM users;

-- DML: Delete a user
DELETE FROM users WHERE id = 1;

-- TCL: Rollback changes
ROLLBACK;

-- DCL: Grant access
GRANT SELECT, INSERT ON users TO 'student'@'localhost';

3.5 MySQL: A Popular Relational DBMS

MySQL is an open-source relational database management system (RDBMS) widely used in web applications. It is highly compatible with PHP, Laravel, Node.js, and integrates well with cloud hosting platforms.

  • Follows SQL standards with additional features
  • Supports large datasets and optimized indexing
  • Ideal for small and large-scale applications
  • Can be managed using GUI tools like MySQL Workbench or CLI

4. Getting Started with MySQL

MySQL is an open-source Relational Database Management System (RDBMS) based on Structured Query Language (SQL). It is widely used in web applications to store, retrieve, and manipulate data in a structured way using tables, columns, and rows.

Understanding how to use MySQL is essential for any backend or full-stack developer. This section introduces the fundamentals: how to define databases and tables, insert and retrieve information, and apply good design practices.

4.1. Installing MySQL

You can work with MySQL in different environments:

  • Local Installation: You can install MySQL directly on your machine. Options include:
    • MySQL Installer (Windows/Mac)
    • XAMPP – includes Apache, MySQL, and phpMyAdmin
    • Docker (using mysql container)
  • Cloud Platforms: Cloud solutions like AWS RDS, Google Cloud SQL, or PlanetScale offer scalable MySQL databases.
  • Browser-based tools: Tools like db-fiddle or SQL Fiddle let you practice without installation.

4.2. What Is a Relational Database?

A relational database stores data in related tables, using rows (records) and columns (fields). Each table represents one type of entity: for example, a users table for people, and a tasks table for their to-do items.

These tables are linked through relationships, typically using PRIMARY KEY and FOREIGN KEY constraints.

4.3. Creating a Database and Tables

Let’s walk through creating a basic project management database called task_manager.

-- Step 1: Create a new database
CREATE DATABASE task_manager;

-- Step 2: Use the new database
USE task_manager;

-- Step 3: Create a 'users' table
CREATE TABLE users (
  id INT AUTO_INCREMENT PRIMARY KEY,
  name VARCHAR(100) NOT NULL,
  email VARCHAR(100) NOT NULL UNIQUE,
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

-- Step 4: Create a 'tasks' table
CREATE TABLE tasks (
  id INT AUTO_INCREMENT PRIMARY KEY,
  title VARCHAR(255) NOT NULL,
  description TEXT,
  status ENUM('pending', 'in_progress', 'completed') DEFAULT 'pending',
  user_id INT,
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
  FOREIGN KEY (user_id) REFERENCES users(id)
);

Explanation:

  • AUTO_INCREMENT: Automatically increases the ID for each new record.
  • VARCHAR: Used for text fields of variable length.
  • ENUM: Limits a field to specific values (great for status fields).
  • FOREIGN KEY: Creates a link between tasks.user_id and users.id.

4.4. Inserting Records

We can now populate our tables with example data:

-- Insert users
INSERT INTO users (name, email) VALUES
('Alice Johnson', 'alice@example.com'),
('Bob Smith', 'bob@example.com');

-- Insert tasks
INSERT INTO tasks (title, description, status, user_id) VALUES
('Buy groceries', 'Milk, eggs, bread', 'pending', 1),
('Finish report', 'Due by Monday', 'in_progress', 2),
('Clean workspace', 'Desk and shelves', 'completed', 1);

Each task is linked to a user through the user_id field.

4.5. Querying the Database

Now we can retrieve and manipulate data using SELECT statements:

-- View all users
SELECT * FROM users;

-- View all tasks for Alice
SELECT title, status
FROM tasks
WHERE user_id = 1;

-- Join tasks and users
SELECT tasks.title, tasks.status, users.name
FROM tasks
JOIN users ON tasks.user_id = users.id;

-- Count tasks by status
SELECT status, COUNT(*) AS count
FROM tasks
GROUP BY status;

-- Show most recent task
SELECT * FROM tasks
ORDER BY created_at DESC
LIMIT 1;

4.6. Updating and Deleting Records

You can also modify or remove data from the database:

-- Update a task's status
UPDATE tasks
SET status = 'completed'
WHERE id = 2;

-- Delete a task
DELETE FROM tasks
WHERE id = 3;

4.7. Good Database Practices

Designing and managing a database goes beyond writing correct SQL queries. To ensure efficiency, reliability, and maintainability, it's essential to follow a set of good practices. This section highlights fundamental guidelines for creating well-structured databases, preventing common errors, and promoting scalability. Applying these principles early on helps avoid performance issues and supports long-term development.

  • Normalize your tables: Avoid duplication by separating concerns into different tables.
  • Use meaningful names: Tables and columns should describe the data they contain.
  • Always define primary and foreign keys: This keeps data structured and prevents orphaned records.
  • Add indexes: On frequently searched fields for faster performance.
  • Backup regularly: Always keep data secure and backed up—especially in production environments.
