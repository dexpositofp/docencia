Presentación de la asignatura

Base sólida en HTML, CSS y ecosistema web para crear interfaces accesibles, rápidas y mantenibles.

¡Bienvenidas y bienvenidos al módulo de Lenguaje de Marcas! En esta asignatura construiremos la base técnica y conceptual para maquetar y dar estilo a interfaces web de calidad. Trabajaremos con HTML (estructura), CSS (presentación) y un primer contacto con JavaScript (interacción), siempre con foco en accesibilidad, rendimiento y buenas prácticas.

¿Qué esperar? Material autodidáctico, laboratorios guiados, feedback rápido y proyectos aplicados.

1. Objetivos del curso

  • Diferenciar roles de HTML, CSS y JS y aplicarlos de forma semántica y mantenible.
  • Dominar CSS moderno: cascada, especificidad, box model, layout con Flexbox y Grid, responsive y variables.
  • Construir componentes accesibles (teclado, :focus-visible, contraste WCAG) y rápidos.
  • Integrar herramientas y metodologías (BEM, organización por capas, preprocesadores) en proyectos reales.

2. Metodología y dinámica

  • Clases prácticas con demostraciones y laboratorios paso a paso.
  • Aprendizaje activo: retos cortos, ejercicios con solución y rúbricas claras.
  • Proyecto por unidad con entregas parciales (iteración y feedback).
  • Autonomía: material de estudio y guías “hazlo tú” para avanzar a tu ritmo.

Consejo: Se recomienda utilizar un editor (VS Code) y probar todo usando devtools.

3. Unidades

UD1 Lenguajes de Marcas

Explorarás qué es un lenguaje de marcas y para qué se usa en la web y en el intercambio de datos. Trabajarás con XML (estructura, atributos, namespaces), su validación con DTD y XSD, y conocerás Markdown para documentación técnica. Practicarás con pequeños documentos bien formados, crearás un esquema sencillo para validar datos y aprenderás a usar validador online.

Entregable: Colección de ejemplos XML válidos + una breve guía en Markdown.

Ir a la unidad

UD2 HTML

Construirás la estructura de páginas accesibles usando las etiquetas semánticas de HTML5 (<header>, <main>, <section>, <article>, <nav>, <footer>). Verás imágenes, listas, tablas, enlaces internos/externos, rutas y formularios (inputs HTML5, validación nativa). Practicarás buenas prácticas de accesibilidad (texto alternativo, jerarquía de encabezados, labels) y validarás el documento con el validador del W3C.

Entregable: Mini-sitio multipágina con navegación coherente y formulario funcional.

Ir a la unidad

UD3 CSS

Dominarás la cascada, herencia y especificidad, los selectores clásicos y modernos (:is(), :where(), :has()), y el box model. Pasarás al layout moderno con Flexbox y Grid (componentes, áreas, auto-fit), responsive con media queries, tipografía fluida (clamp()) y variables CSS. Añadirás transiciones y animaciones respetando prefers-reduced-motion.

Entregable: Landing responsive con componentes reutilizables y efectos de interacción.

Ir a la unidad

UD4 JavaScript

Introducirás interactividad con JavaScript: tipos y operadores, funciones y módulos, DOM y eventos, fetch para consumir datos y renderizado dinámico. Verás patrones básicos (delegación de eventos, separación de lógica y vista) y buenas prácticas de accesibilidad (gestión del foco, ARIA mínima).

Entregable: Página que consume un JSON y pinta tarjetas filtrables con interacción.

Ir a la unidad

Proyecto

Aplicarás todo lo aprendido en un proyecto completo: definición de contenido, diseño visual y responsive, accesibilidad básica, interactividad con JS y, si procede, plantillas en servidor. Trabajarás con control de versiones, organización del CSS (BEM y capas) y despliegue estático.

Entregables: repositorio limpio documentado, demo desplegada y retrospectiva técnica breve.

4. Evaluación

La calificación por unidad se compone de:

  • Prueba objetiva (60%): Individual, mínima 5 para promediar.
  • Práctica evaluable (10%): Individual, mínima 5 para promediar.
  • Proyecto/prácticas (30%): Entregas y defensa/validación individual obligatoria.Nota mínima 5 para promediar.

Importante: para poder tener en cuenta la nota del proyecto es imprescindible superar la validación individual del trabajo realizado. Si no se supera, el proyecto calificará con 0.

¡A tener en cuenta! para cada prueba que se realice, se os puede pedir en cualquier momento una prueba de validación para verificar el trabajo realizado. Si no se supera, dicha prueba se calificará con 0.

La planificación detallada (resultados de aprendizaje, actividades y ponderaciones) se recoge en la programación del curso.

6. Normas del aula y entregas

  • Respeta las fechas de entrega. No serán aceptadas las entregas fuera de plazo.
  • El código debe ser propio. Cita fuentes y referencia librerías/recursos externos. En cualquier momento se puede solicitar una validación individual para verificar autoría y entendimiento de la entrega.
  • Accesibilidad y semántica cuentan: validadores y chequeos de contraste forman parte de las rúbricas.
