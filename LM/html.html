HTML

Estructura y el significado del contenido de una página web

Introducción a HTML

HTML (HyperText Markup Language) es el estándar para estructurar documentos en la web. En combinación con CSS (presentación) y JavaScript (interactividad), conforma la base tecnológica del front-end.

¿Qué es HTML?

Es un lenguaje de marcado: no programa, sino que etiqueta contenido para describir su función (párrafos, listas, imágenes, tablas, formularios…). Los navegadores interpretan estas etiquetas para construir el árbol del documento (DOM) y renderizarlo.

Significado de las siglas

  • Hyper: hipervínculos que conectan recursos entre sí.
  • Text: el contenido que se presenta (texto, imagen, multimedia).
  • Markup: marcado semántico que da estructura y significado.
  • Language: conjunto formal de elementos y atributos con sintaxis definida.

Historia (muy breve) y HTML5

Desde 1991 HTML ha evolucionado hasta HTML5, que incorpora etiquetas semánticas (header, nav, main, section, article…), formularios enriquecidos y APIs modernas del navegador (multimedia, canvas, etc.).

Ejemplo mínimo

<!DOCTYPE html>
<html lang="es">
<head>...</head>
<body>
  <h1>¡Hola, mundo!</h1>
  <p>Primer documento HTML.</p>
</body>
</html>

Separación de responsabilidades: HTML (estructura y significado), CSS (presentación), JS (comportamiento). Esta separación facilita accesibilidad, mantenimiento y pruebas.

Sintaxis y etiquetas básicas

Un elemento HTML consta de etiqueta de apertura, contenido y (normalmente) etiqueta de cierre. Los atributos añaden metadatos o comportamientos. Usa etiquetas semánticas siempre que sea posible y deja <div>/<span> para cuando no exista una opción mejor.

Texto, títulos y enlaces

  • <h1..h6>: jerarquía de encabezados. Normalmente un <h1> por página o por contenido principal.
  • <p>: párrafos; <strong>/<em> para énfasis (semántico, no visual).
  • <a href>: enlaces con texto significativo (evita “clic aquí”). Si abres en nueva pestaña, añade rel="noopener noreferrer".
<h1>Guía de Viaje</h1>
<p>Consulta los <a href="#consejos">consejos previos</a> antes de reservar.</p>
<p>Las ofertas están <strong>disponibles</strong> hasta fin de mes.</p>

Listas: no ordenadas, ordenadas y de definiciones

Elige la lista según el significado, no por la apariencia:

  • No ordenada (<ul>): conjunto sin orden intrínseco.
  • Ordenada (<ol>): pasos o jerarquía con orden.
  • Definiciones (<dl>): términos y descripciones.
<h2>Checklist de viaje</h2>
<ul>
  <li>Pasaporte</li>
  <li>Tarjeta sanitaria</li>
  <li>Adaptador de enchufe</li>
</ul>

<h2>Pasos para reservar</h2>
<ol>
  <li>Elegir destino</li>
  <li>Comparar vuelos</li>
  <li>Confirmar alojamiento</li>
</ol>

<h2>Términos clave</h2>
<dl>
  <dt>Check-in</dt><dd>Registro de entrada al alojamiento.</dd>
  <dt>Escala</dt><dd>Parada intermedia entre vuelos.</dd>
</dl>

Consejo: anidar listas está bien si la jerarquía lo pide, pero evita niveles profundos (pierden claridad).

Imágenes (alt, dimensiones, responsive)

  • <img src alt>: alt describe la imagen; si es decorativa, usa alt="".
  • width/height ayudan a evitar saltos de contenido.
  • Responsive: usa srcset/sizes o <picture> para formatos alternativos.
<figure>
  <img
    src="img/playa-800.jpg"
    srcset="img/playa-400.jpg 400w, img/playa-800.jpg 800w, img/playa-1200.jpg 1200w"
    sizes="(max-width: 600px) 90vw, 800px"
    width="800" height="533"
    alt="Playa con cielo despejado y sombrillas">
  <figcaption>Vista de la playa al atardecer.</figcaption>
</figure>

Alt correcto: qué es la imagen y por qué está ahí. Evita “imagen de…”. Describe la información relevante para el contexto.

Rutas y URLs en src/href

  • Relativas (p. ej., ../img/logo.svg).
  • Absolutas (p. ej., https://example.com).
  • Fragmentos (p. ej., #contacto).

Contenedores neutros (usar con criterio)

  • <div>: bloque sin semántica (para agrupar).
  • <span>: en línea sin semántica (para fragmentos dentro de texto).

Regla de oro: si existe una etiqueta semántica específica para tu caso (por ejemplo, <header> en lugar de un <div class="header">), úsala.

Etiquetas semánticas de seccionado (HTML5)

Estas etiquetas dan estructura y significado al documento. Mejoran accesibilidad, SEO y mantenimiento. No cambian la apariencia por sí mismas (eso lo hace CSS), pero crean una “columna vertebral” semántica clara.

<main>

Contenido principal único de la página. Debe aparecer una sola vez y no debe anidar otras regiones de landmark como header o footer del sitio.

<header> y <footer>

  • De página: cabecera y pie globales del sitio.
  • De sección o artículo: puedes tener headers/footers locales dentro de section/article para títulos, metadatos, navegación local, etc.

<nav>

Bloque de navegación con enlaces principales o secundarios. Puede existir más de uno (nav global, nav de migas, nav de sección).

<section> vs <article>

  • <article>: unidad de contenido independiente y reutilizable (post, tarjeta de producto, noticia). Tiene sentido fuera de contexto.
  • <section>: agrupa contenido relacionado con un encabezado propio, pero que necesita el contexto de la página.

<aside>

Contenido tangencial al principal (barras laterales, promos, glosarios). En lectores de pantalla se anuncia como complemento.

<figure> y <figcaption>

Para imágenes, gráficos, fragmentos de código o tablas que requieren título o explicación.

<time datetime>

Fecha/hora legible por máquina (útil para SEO, microformatos y ordenación).

Patrón de página semántico

<header>
  <h1>Nombre del sitio</h1>
  <nav>
    <ul><li><a href="/">Inicio</a></li>...</ul>
  </nav>
</header>

<main>
  <article>
    <header>
      <h2>Título del artículo</h2>
      <p><time datetime="2025-10-18">18 oct 2025</time> — Por Ana</p>
    </header>

    <section>
      <h3>Introducción</h3>
      <p>...</p>
    </section>

    <section>
      <h3>Desarrollo</h3>
      <figure>
        <img src="img/diagrama.svg" alt="Flujo de trabajo del sistema">
        <figcaption>Diagrama del flujo de trabajo.</figcaption>
      </figure>
    </section>

    <footer>
      <p>Etiquetas: <a href="/tags/html">HTML</a>, <a href="/tags/accesibilidad">Accesibilidad</a></p>
    </footer>
  </article>

  <aside>
    <h2>Relacionado</h2>
    <ul>
      <li><a href="/articulo-1">Buenas prácticas de ALT</a></li>
    </ul>
  </aside>
</main>

<footer>
  <small>© 2025 — Nombre del centro</small>
</footer>

Encabezados y outline: usa niveles de h1..h6 de forma jerárquica. Un patrón típico es: 1× h1 para el título de la página y luego h2 para secciones, h3 para subsecciones, etc.

Widgets nativos útiles

  • <details> + <summary> para acordeones accesibles sin JS.
  • <progress> / <meter> para representar estados/valores.
  • <dialog> (con algo de JS) para modales accesibles.
<details>
  <summary>Ver requisitos mínimos</summary>
  <ul><li>Navegador moderno</li><li>Conexión estable</li></ul>
</details>

Evita usar <div> con clases tipo header/footer/section cuando existen etiquetas semánticas reales. Mejora la navegación por landmarks de lectores de pantalla y clarifica el DOM.

Principios de diseño web

Guían la experiencia de usuario: usabilidad (flujo claro), consistencia (patrones), legibilidad (tipografía, contraste), accesibilidad (contenido para todos) y rendimiento (tiempos de carga razonables).

  • Navegación: ubicaciones predecibles, feedback visual del estado, menús accesibles.
  • Tipografía: tamaños relativos, alturas de línea generosas, párrafos no excesivamente largos.
  • Color: contraste suficiente (WCAG); no apoyarse solo en el color para transmitir estado.

Consejo: diseña primero el flujo de información; el estilo debe potenciarlo, no ocultarlo.

Usabilidad y accesibilidad

Usabilidad es eficacia + eficiencia + satisfacción. Accesibilidad significa que el contenido es usable por personas con diferentes capacidades (teclado, lectores de pantalla, visión reducida, etc.).

  • Usa landmarks (header/nav/main/footer), encabezados jerárquicos y alt adecuados.
  • En formularios: <label for> + id, required y mensajes asociados (aria-describedby).
  • Gestión del foco: secuencia lógica, focus visible y roles ARIA solo cuando falte semántica nativa.
<a class="skip-link" href="#main">Saltar al contenido</a>

Formularios

Recopilan datos del usuario. Un buen formulario combina semántica, accesibilidad, validación en el cliente (HTML5) y en el servidor.

Estructura básica

La etiqueta <form> encapsula los controles. Usa siempre name para enviar datos y label para accesibilidad.

<form action="/submit" method="post" novalidate>
  <div>
    <label for="nombre">Nombre</label>
    <input id="nombre" name="nombre" type="text" required autocomplete="name">
  </div>
  <div>
    <label for="email">Email</label>
    <input id="email" name="email" type="email" required autocomplete="email">
  </div>
  <button type="submit">Enviar</button>
</form>

method: get para lecturas o filtros (parámetros en URL); post para envíos. action: ruta del procesamiento.

Atributos y estados útiles

  • required, min, max, minlength, maxlength, step, pattern
  • Pseudoclases CSS: :required, :optional, :valid, :invalid, :in-range, :out-of-range
  • Texto de ayuda con aria-describedby y gestionado por id.
<label for="tel">Teléfono (9 dígitos)</label>
<input id="tel" name="tel" type="tel" pattern="[0-9]{9}" required
       aria-describedby="ayuda-tel">
<small id="ayuda-tel" class="muted">Solo números, sin espacios.</small>

Agrupación y ayudas

Para estructurar y mejorar accesibilidad:

  • <fieldset> agrupa controles relacionados + <legend> describe el grupo.
  • <optgroup> organiza opciones dentro de <select>.
  • <datalist> sugiere valores a un <input list>.
<fieldset>
  <legend>Datos personales</legend>
  <label for="nombre">Nombre</label>
  <input id="nombre" name="nombre" type="text">
  <label for="ciudad">Ciudad</label>
  <input list="ciudades" id="ciudad" name="ciudad">
  <datalist id="ciudades">
    <option value="Madrid">
    <option value="Valencia">
  </datalist>
</fieldset>

Validación en el navegador

HTML5 implementa validación nativa: formatos (email, url), patrones (regex), requeridos, rangos, etc. Es complementaria a la validación en el servidor.

<input type="email" name="correo" required>
<input type="text" name="telefono" pattern="[0-9]{9}" required>

Importante: nunca confíes solo en la validación del cliente; valida y sanea siempre en el servidor.

Ejercicios prácticos de formularios

  • Registro con nombre, apellidos, email, contraseña y confirmación (con required y ayuda de formato).
  • Reserva de cita: fecha, hora y lista de servicios (select/radio).
  • Subida de imagen: acepta solo .jpg y .png (atributo accept).
  • Encuesta con checkbox y radio y un campo de comentarios.
  • Contacto con validación de email y teléfono por patrón.

Tablas y multimedia

Tablas accesibles

Úsalas para datos tabulares, no para maquetación. Incluye <caption>, cabeceras y scope adecuado.

<table>
  <caption>Resultados trimestrales</caption>
  <thead>
    <tr><th scope="col">Mes</th><th scope="col">Ventas</th></tr>
  </thead>
  <tbody>
    <tr><th scope="row">Enero</th><td>120k</td></tr>
  </tbody>
</table>

Multimedia en HTML5

  • <audio> y <video> con controls, preload y pistas de subtítulos (<track kind="captions">).
  • <canvas> para gráficos via JS; <svg> para vectoriales semánticos.
<video width="480" controls preload="metadata">
  <source src="movie.mp4" type="video/mp4">
  <track src="sub.vtt" kind="captions" srclang="es" label="Español">
  Tu navegador no soporta video.
</video>
<canvas id="graf" width="240" height="120"></canvas>
<script>
  const ctx = document.getElementById('graf').getContext('2d');
  ctx.fillRect(10, 10, 80, 40);
</script>

Validación de documentos HTML

La validación ayuda a detectar errores de marcado y a mejorar la compatibilidad. Incorpórala al flujo de trabajo (validadores, linters, pruebas en varios navegadores/lectores de pantalla).

Checklist rápido: lang y charset correctos, semántica adecuada, contrastes suficientes, imágenes con alt, formularios etiquetados, y sin errores en el validador W3C.

Restricciones que impone XHTML

XHTML es HTML reescrito como XML. Exige documentos bien formados: etiquetas en minúscula, cierre obligatorio, atributos entre comillas y un único elemento raíz. Su enfoque estricto favoreció interoperabilidad y herramientas XML; hoy HTML5 domina, pero las reglas de “bien formado” siguen siendo útiles como disciplina formal.

Ejercicios básicos de HTML

  1. Presentación personal

    Incluye un <h1> con tu nombre, un párrafo con intereses, y una lista de favoritos.

  2. Estructura base

    Crea un documento mínimo con <!DOCTYPE>, <html>, <head> y <body>, usando lang="es", charset y viewport.

  3. Enlace e imagen

    Añade un <a> con texto significativo y una <img> centrada con alt descriptivo.

  4. Formulario básico

    Nombre, email, comentarios y botón de envío, con label y required.

  5. Tabla simple

    3×3 con cabeceras en la primera fila y un <caption>.
