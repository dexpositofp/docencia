Anatomía de un documento HTML5

HTML define la estructura y el significado del contenido. Un documento mínimo debería incluir una declaración de tipo, el idioma, una cabecera con metadatos y el contenido principal en <body>:

<!DOCTYPE html>
<html lang="es">
<head>
  <meta charset="utf-8">
  <meta name="viewport" content="width=device-width, initial-scale=1">
  <title>Título de la página</title>
  <meta name="description" content="Descripción breve para buscadores y redes.">
  <link rel="icon" href="/favicon.ico">
  <link rel="stylesheet" href="styles.css">
</head>
<body>
  <main>Contenido principal</main>
</body>
</html>

Buenas prácticas en <head>: usa lang correcto, charset="utf-8", viewport para responsive, title conciso y meta description informativa. Añade rel="preconnect"/preload solo cuando mejoren rendimiento de forma medible.

Introducción a HTML

HTML (HyperText Markup Language) es el estándar para estructurar documentos en la web. En combinación con CSS (presentación) y JavaScript (interactividad), conforma la base tecnológica del front-end.

¿Qué es HTML?

Es un lenguaje de marcado: no programa, sino que etiqueta contenido para describir su función (párrafos, listas, imágenes, tablas, formularios…). Los navegadores interpretan estas etiquetas para construir el árbol del documento (DOM) y renderizarlo.

Significado de las siglas

  • Hyper: hipervínculos que conectan recursos entre sí.
  • Text: el contenido que se presenta (texto, imagen, multimedia).
  • Markup: marcado semántico que da estructura y significado.
  • Language: conjunto formal de elementos y atributos con sintaxis definida.

Historia (muy breve) y HTML5

Desde 1991 HTML ha evolucionado hasta HTML5, que incorpora etiquetas semánticas (header, nav, main, section, …), formularios enriquecidos (inputs con validación), y APIs modernas del navegador (multimedia, canvas, etc.).

Ejemplo mínimo

<!DOCTYPE html>
<html lang="es">
<head>...</head>
<body>
  <h1>¡Hola, mundo!</h1>
  <p>Primer documento HTML.</p>
</body>
</html>

Separación de responsabilidades: HTML (estructura y significado), CSS (presentación), JS (comportamiento). Esta separación facilita accesibilidad, mantenimiento y pruebas.

Sintaxis y etiquetas básicas

HTML utiliza elementos (etiquetas) y atributos. Algunos de los más frecuentes:

  • <h1..h6> jerarquía de encabezados (no saltes niveles sin motivo).
  • <p> párrafos; <strong>/<em> para énfasis semántico.
  • <a href> enlaces con texto significativo (evita “clic aquí”).
  • <img src alt> imágenes con alt descriptivo (o alt="" si decorativas).
  • <ul>/<ol> listas; <li> elementos de lista.
  • <div>/<span> contenedores sin semántica (úsalos con criterio).

Bloques semánticos de alto nivel (HTML5)

  • <header>, <nav>, <main>, <section>, <article>, <aside>, <footer>
  • <figure> y <figcaption> para contenido ilustrado.
  • <time> para fechas con datetime.

Ejemplo de estructura semántica

<header>...</header>
<main>
  <article>
    <h1>Título del artículo</h1>
    <section><h2>Contexto</h2><p>...</p></section>
    <section><h2>Contenido</h2><p>...</p></section>
    <footer><small>Autoría y fecha</small></footer>
  </article>
</main>
<footer>...</footer>

Errores típicos: imágenes sin alt, usar <br> para espaciado, enlaces con texto genérico, abuso de div cuando existen etiquetas semánticas adecuadas.

Principios de diseño web

Guían la experiencia de usuario: usabilidad (flujo claro), consistencia (patrones), legibilidad (tipografía, contraste), accesibilidad (contenido para todos) y rendimiento (tiempos de carga razonables).

  • Navegación: ubicaciones predecibles, feedback visual del estado, menús accesibles.
  • Tipografía: tamaños relativos, alturas de línea generosas, párrafos no excesivamente largos.
  • Color: contraste suficiente (WCAG); no apoyarse solo en el color para transmitir estado.

Consejo: diseña primero el flujo de información; el estilo debe potenciarlo, no ocultarlo.

Usabilidad y accesibilidad

Usabilidad es eficacia + eficiencia + satisfacción. Accesibilidad significa que el contenido es usable por personas con diferentes capacidades (teclado, lectores de pantalla, visión reducida, etc.).

  • Usa landmarks (header/nav/main/footer), encabezados jerárquicos y alt adecuados.
  • En formularios: <label for> + id, required y mensajes asociados (aria-describedby).
  • Gestión del foco: secuencia lógica, focus visible y roles ARIA solo cuando falte semántica nativa.

Formularios

Recopilan datos del usuario. Un buen formulario combina semántica, accesibilidad, validación en el cliente (HTML5) y en el servidor.

Estructura básica

La etiqueta <form> encapsula los controles. Usa siempre name para enviar datos y label para accesibilidad.

<form action="/submit" method="post" novalidate>
  <div>
    <label for="nombre">Nombre</label>
    <input id="nombre" name="nombre" type="text" required autocomplete="name">
  </div>
  <div>
    <label for="email">Email</label>
    <input id="email" name="email" type="email" required autocomplete="email">
  </div>
  <button type="submit">Enviar</button>
</form>

method: get para lecturas o filtros (parámetros en URL); post para envío de datos. action: ruta del procesamiento.

Atributos y estados útiles

  • required, min, max, minlength, maxlength, step, pattern
  • Pseudoclases CSS: :required, :optional, :valid, :invalid, :in-range, :out-of-range
  • Texto de ayuda con aria-describedby y gestionado por id.
<label for="tel">Teléfono (9 dígitos)</label>
<input id="tel" name="tel" type="tel" pattern="[0-9]{9}" required
       aria-describedby="ayuda-tel">
<small id="ayuda-tel" class="muted">Solo números, sin espacios.</small>

Agrupación y ayudas

Para estructurar y mejorar accesibilidad:

  • <fieldset> agrupa controles relacionados + <legend> describe el grupo.
  • <optgroup> organiza opciones dentro de <select>.
  • <datalist> sugiere valores a un <input list>.
<fieldset>
  <legend>Datos personales</legend>
  <label for="nombre">Nombre</label>
  <input id="nombre" name="nombre" type="text">
  <label for="ciudad">Ciudad</label>
  <input list="ciudades" id="ciudad" name="ciudad">
  <datalist id="ciudades">
    <option value="Madrid">
    <option value="Valencia">
  </datalist>
</fieldset>

Validación en el navegador

HTML5 implementa validación nativa: formatos (email, url), patrones (regex), requeridos, rangos, etc. Es complementaria a la validación en el servidor.

<input type="email" name="correo" required>
<input type="text" name="telefono" pattern="[0-9]{9}" required>

Importante: nunca confíes solo en la validación del cliente; valida y sanea siempre en el servidor.

Ejercicios prácticos de formularios

  • Registro con nombre, apellidos, email, contraseña y confirmación (con required y ayuda de formato).
  • Reserva de cita: fecha, hora y lista de servicios (select/radio).
  • Subida de imagen: acepta solo .jpg y .png (atributo accept).
  • Encuesta con checkbox y radio y un campo de comentarios.
  • Contacto con validación de email y teléfono por patrón.

Tablas y multimedia

Tablas accesibles

Úsalas para datos tabulares, no para maquetación. Incluye <caption>, cabeceras y scope adecuado.

<table>
  <caption>Resultados trimestrales</caption>
  <thead>
    <tr><th scope="col">Mes</th><th scope="col">Ventas</th></tr>
  </thead>
  <tbody>
    <tr><th scope="row">Enero</th><td>120k</td></tr>
  </tbody>
</table>

Multimedia en HTML5

  • <audio> y <video> con controls, preload y pistas de subtítulos (<track kind="captions">).
  • <canvas> para gráficos via JS; <svg> para vectoriales semánticos.
<video width="480" controls preload="metadata">
  <source src="movie.mp4" type="video/mp4">
  <track src="sub.vtt" kind="captions" srclang="es" label="Español">
  Tu navegador no soporta video.
</video>
<canvas id="graf" width="240" height="120"></canvas>
<script>
  const ctx = document.getElementById('graf').getContext('2d');
  ctx.fillRect(10, 10, 80, 40);
</script>

Validación de documentos HTML

La validación ayuda a detectar errores de marcado y a mejorar la compatibilidad. Incorpórala al flujo de trabajo (validadores, linters, pruebas en varios navegadores/lectores de pantalla).

Checklist rápido: lang y charset correctos, semántica adecuada, contrastes suficientes, imágenes con alt, formularios etiquetados, y sin errores en el validador W3C.

Restricciones que impone XHTML

XHTML es HTML reescrito como XML. Exige documentos bien formados: etiquetas en minúscula, cierre obligatorio, atributos entre comillas y un único elemento raíz. Su enfoque estricto favoreció interoperabilidad y herramientas XML; hoy HTML5 domina, pero las reglas de “bien formado” siguen siendo útiles como disciplina formal.

Ejercicios básicos de HTML

  1. Presentación personal

    Incluye un <h1> con tu nombre, un párrafo con intereses, y una lista de favoritos.

  2. Estructura base

    Crea un documento mínimo con <!DOCTYPE>, <html>, <head> y <body>, usando lang="es", charset y viewport.

  3. Enlace e imagen

    Añade un <a> con texto significativo y una <img> centrada con alt descriptivo.

  4. Formulario básico

    Nombre, email, comentarios y botón de envío, con label y required.

  5. Tabla simple

    3×3 con cabeceras en la primera fila y un <caption>.
