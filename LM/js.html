JavaScript

¿Qué es JavaScript?

JavaScript es el lenguaje de programación de la web.

Si HTML es la estructura de una página web (como el esqueleto de un edificio) y CSS es el diseño visual (como la pintura y decoración), JavaScript es el comportamiento, lo que hace que las páginas sean interactivas y dinámicas.

Cuando haces clic en un botón y aparece un menú, cuando escribes en un buscador y te sugiere resultados, cuando actualizas tu feed de redes sociales sin recargar la página... todo eso es JavaScript en acción.

Dato importante: Aunque comparten parte del nombre, Java y JavaScript son lenguajes completamente diferentes. Solo comparten algunas letras, pero son cosas distintas.

¿Por qué JavaScript es especial?

JavaScript es un lenguaje de programación que entienden los navegadores web de forma nativa. Esto significa que:

  • No necesitas instalar nada especial para ejecutarlo
  • Funciona en cualquier navegador (Chrome, Firefox, Safari, Edge...)
  • Funciona en cualquier sistema operativo (Windows, Mac, Linux, móviles...)
  • Es el lenguaje más usado en desarrollo web

¿Dónde se ejecuta JavaScript?

Originalmente, JavaScript solo funcionaba en el navegador. Pero hoy en día, JavaScript se ejecuta en muchos sitios:

Entorno Descripción Ejemplo
Navegador Frontend - Lo que ve el usuario Validar formularios, mostrar/ocultar elementos, animaciones
Servidor Backend - Con Node.js, Deno, Bun APIs REST, bases de datos, autenticación
Móvil Apps nativas - React Native, Ionic Instagram, Discord (usan React Native)
Escritorio Aplicaciones de escritorio - Electron Visual Studio Code, Slack, Discord

En esta evaluación nos centraremos en JavaScript en el navegador, que es la base fundamental. Una vez domines JavaScript en el navegador, aprender Node.js u otros entornos será mucho más fácil.

JavaScript vs Java: Diferencias importantes

Como ya sabéis programar en Java, es importante entender las diferencias clave:

Característica Java JavaScript
Tipado Estático (tipos fijos) Dinámico (tipos flexibles)
Declaración de variables int edad = 20; let edad = 20;
Compilación Compilado a bytecode Interpretado (ejecutado directamente)
Ejecución JVM (Java Virtual Machine) Navegador o Node.js
Orientación a objetos Fuertemente orientado a objetos (todo son clases) Multi-paradigma (funcional + objetos + prototipos)
Sintaxis Más verbosa y estricta Más flexible y permisiva

Ejemplo comparativo:

// JAVA
public class Main {
    public static void main(String[] args) {
        String nombre = "Ana";
        int edad = 20;
        System.out.println("Hola " + nombre + ", tienes " + edad + " años");
    }
}

// JAVASCRIPT
let nombre = "Ana";
let edad = 20;
console.log(`Hola ${nombre}, tienes ${edad} años`);

Como ves, JavaScript es generalmente más conciso. No necesitas clases ni métodos principales para ejecutar código simple.

Tu primer programa en JavaScript

Vamos a crear nuestro primer programa. Crea un archivo llamado primero.html con este contenido:

<!DOCTYPE html>
<html lang="es">
<head>
  <meta charset="UTF-8">
  <title>Mi primer JavaScript</title>
</head>
<body>
  <h1>Mi primer programa en JavaScript</h1>
  <p>Abre la consola del navegador (F12) para ver el resultado</p>

  <script>
    // Esto es un comentario en JavaScript
    console.log("¡Hola mundo desde JavaScript!");

    // También podemos hacer operaciones
    console.log(2 + 2);

    // Y mostrar alertas
    alert("¡Bienvenido a JavaScript!");
  </script>
</body>
</html>

Qué hace este código:

  • console.log() → Muestra mensajes en la consola del navegador (similar a System.out.println() en Java)
  • alert() → Muestra una ventana emergente al usuario
  • // → Comentarios de una línea (igual que en Java)

Para ejecutarlo:

  1. Guarda el archivo como primero.html
  2. Ábrelo con tu navegador (doble clic)
  3. Presiona F12 (o clic derecho → Inspeccionar → Consola)
  4. Verás los mensajes en la consola

⚠️ Consejo importante: La consola del navegador (F12) será tu mejor amiga mientras aprendes JavaScript. Úsala constantemente para probar código y ver los resultados.

¿Dónde escribir el código JavaScript?

Tienes tres opciones para incluir JavaScript en una página HTML:

1. En línea (inline) - ❌ NO recomendado

<button onclick="alert('Hola')">Clic aquí</button>

No lo uses: Mezcla HTML con JavaScript, dificulta el mantenimiento.

2. En el mismo archivo HTML (interno)

<script>
  console.log("JavaScript interno");
</script>

Útil para: Pruebas rápidas o scripts muy pequeños.

3. En archivo externo - ✅ RECOMENDADO

<!-- En el HTML -->
<script src="app.js" defer></script>

/* En app.js */
console.log("JavaScript externo");

Mejor opción: Separa estructura (HTML) de comportamiento (JS). Más organizado y reutilizable.

💡 El atributo defer: Hace que el script se ejecute después de que el HTML esté completamente cargado. Siempre úsalo cuando tu JavaScript necesite acceder a elementos del DOM.

Variables

Como en otros lenguajes, utilizamos las variables para guardamos información. En programación, necesitamos almacenar datos en la memoria del ordenador para poder trabajar con ellos: nombres, números, resultados de cálculos, etc.

En Java, cuando declaras una variable, debes especificar su tipo: 

// JAVA
int edad = 20;
String nombre = "Ana";
boolean estudiante = true;

En JavaScript, no necesitas especificar el tipo. JavaScript es de tipado dinámico, lo que significa que la variable puede contener cualquier tipo de dato y puede cambiar de tipo durante la ejecución: 

// JAVASCRIPT
let edad = 20;           // Ahora es un número
edad = "veinte";         // Ahora es un texto - ¡Permitido! (aunque no recomendado)
let nombre = "Ana";
let estudiante = true;

let: Variables que pueden cambiar

let se usa para declarar variables cuyo valor puede modificarse más adelante en el programa: 

let edad = 20;
console.log(edad);    // 20

edad = 21;            // ✅ Permitido: let puede cambiar
console.log(edad);    // 21

edad = edad + 1;      // También válido
console.log(edad);    // 22

¿Cuándo usar let?

  • Contadores que se incrementan: let contador = 0;
  • Variables que acumulan resultados: let suma = 0;
  • Valores que cambian en bucles: let i = 0;
  • Datos que se actualizan: let edadUsuario = 18;

const: Constantes que NO pueden cambiar

const (de "constante") se usa para valores que NO deben modificarse una vez asignados. Es como una caja cerrada con candado: 

const PI = 3.1416;
console.log(PI);      // 3.1416

PI = 3.2;             // ❌ ERROR: no puedes cambiar una constante
// TypeError: Assignment to constant variable.

¿Cuándo usar const?

  • Constantes matemáticas: const PI = 3.1416;
  • Configuración que no cambia: const MAX_INTENTOS = 3;
  • Referencias a elementos del DOM: const boton = document.getElementById("btn");
  • Valores que no deben modificarse: const IVA = 0.21;

⚠️ Regla de oro: Usa const por defecto. Solo usa let cuando sepas con certeza que el valor va a cambiar. Esto previene errores accidentales y hace tu código más seguro y predecible.

var: La forma antigua (NO usar)

Antes de ES6 (2015), JavaScript solo tenía var para declarar variables. Pero var tiene comportamientos confusos y problemáticos que pueden causar errores difíciles de detectar: 

var edad = 20;      // ❌ NO USES var en código moderno

// Problemas de var:
console.log(nombre);  // undefined (en lugar de error)
var nombre = "Ana";   // "Hoisting" - comportamiento confuso

// Con let:
console.log(apellido); // ❌ ERROR (correcto)
let apellido = "García";

Resumen simple:

  • const → Usa por defecto (valor NO cambia)
  • let → Usa solo si el valor SÍ cambia
  • var → NUNCA lo uses (forma antigua con problemas)

Reglas para nombrar variables

En JavaScript (como en Java), los nombres de variables deben seguir ciertas reglas:

✅ Permitido:

let nombre = "Ana";
let edad2 = 20;
let _privado = "algo";
let $precio = 100;
let nombreCompleto = "Ana García";  // camelCase - recomendado

❌ NO permitido:

let 2nombre = "Ana";          // No puede empezar con número
let nombre-completo = "Ana";  // No puede tener guiones
let let = "algo";             // No puede ser palabra reservada
let class = "DAW";            // class es palabra reservada

Convenciones de nombres (buenas prácticas):

  • camelCase para variables y funciones: nombreCompleto, calcularTotal
  • MAYÚSCULAS para constantes globales: const IVA = 0.21;
  • Nombres descriptivos: edadUsuario mejor que e o x
  • En inglés o español, pero consistente en todo el proyecto

Ámbito (scope) de las variables

El ámbito (scope) determina dónde puedes usar una variable. Las variables declaradas con let y const tienen ámbito de bloque, lo que significa que solo existen dentro del bloque {} donde se declararon: 

// Ámbito global
const nombre = "Ana";

function saludar() {
  // Ámbito de función
  const apellido = "García";
  console.log(nombre);     // ✅ Accede a variable global
  console.log(apellido);   // ✅ Accede a variable local
}

console.log(nombre);       // ✅ Variable global accesible
console.log(apellido);     // ❌ ERROR: apellido no existe aquí

if (true) {
  const edad = 20;         // Solo existe dentro del if
  console.log(edad);       // ✅ 20
}
console.log(edad);         // ❌ ERROR: edad no existe fuera del if

💡 Buena práctica: Declara las variables en el ámbito más pequeño posible. Esto hace tu código más predecible y evita conflictos de nombres.

Ejemplos prácticos

// Ejemplo 1: Contador en un bucle (necesitamos let)
let contador = 0;
while (contador < 5) {
  console.log(contador);
  contador++;  // El valor cambia, por eso usamos let
}

// Ejemplo 2: Configuración que no cambia (usamos const)
const MAX_INTENTOS = 3;
const COLOR_PRIMARIO = "#3498db";
const NOMBRE_APP = "Mi Aplicación";

// Ejemplo 3: Referencias a elementos (usamos const)
const boton = document.getElementById("miBoton");
const formulario = document.querySelector("form");
// Aunque el elemento en sí pueda cambiar su contenido,
// la referencia al elemento no cambia, por eso const

// Ejemplo 4: Valores que se calculan y no cambian
const PI = 3.1416;
const radio = 5;
const area = PI * radio * radio;  // const porque no cambiará
console.log(area);  // 78.54

Tipos de datos en JavaScript

En Java, los tipos de datos son muy importantes y se deben declarar explícitamente. En JavaScript, aunque no los declaramos, siguen existiendo. JavaScript tiene tipado dinámico, lo que significa que el tipo se determina automáticamente según el valor que asignes.

JavaScript tiene 7 tipos de datos primitivos (valores simples) y 1 tipo complejo (objetos):

1. Number (números)

A diferencia de Java (que tiene int, long, float, double...), JavaScript tiene un solo tipo para TODOS los números: 

// En Java necesitarías int, long, float, double...
// En JavaScript TODO es number

const edad = 20;                    // Entero
const altura = 1.75;                // Decimal
const temperatura = -5;             // Negativo
const grande = 9999999999999;       // Número grande
const preciso = 0.1 + 0.2;          // 0.30000000000000004 (imprecisión de coma flotante)

console.log(typeof edad);           // "number"
console.log(typeof altura);         // "number" (todo es number)

Valores especiales de number:

const infinito = Infinity;          // Infinito
const menosInfinito = -Infinity;    // Infinito negativo
const noEsNumero = NaN;             // Not a Number

// Ejemplos de NaN:
console.log(0 / 0);                 // NaN
console.log("hola" * 3);            // NaN (no puedes multiplicar texto)
console.log(parseInt("abc"));       // NaN (no se puede convertir a número)

// Comprobar NaN:
console.log(isNaN(NaN));            // true
console.log(isNaN(123));            // false

Operaciones matemáticas básicas:

const a = 10;
const b = 3;

console.log(a + b);    // 13 (suma)
console.log(a - b);    // 7  (resta)
console.log(a * b);    // 30 (multiplicación)
console.log(a / b);    // 3.333... (división - siempre devuelve decimal)
console.log(a % b);    // 1  (módulo: resto de la división)
console.log(a ** b);   // 1000 (potencia: 10³)

// En Java: 10 / 3 = 3 (división entera)
// En JavaScript: 10 / 3 = 3.333... (siempre decimal)

2. String (cadenas de texto)

Los strings representan texto. Puedes usar comillas simples, dobles o backticks (para template literals): 

const nombre1 = "Ana";                  // Comillas dobles
const nombre2 = 'Luis';                 // Comillas simples
const saludo = `Hola, ${nombre1}`;      // Template literal (backticks)

// En Java:
// String nombre = "Ana";
// System.out.println("Hola " + nombre);

// En JavaScript (moderno):
const edad = 20;
console.log(`Hola ${nombre1}, tienes ${edad} años`);
// "Hola Ana, tienes 20 años"

Template literals (plantillas de texto):

Los template literals (con backticks: `) son una característica muy potente de JavaScript moderno. Permiten:

  • Interpolar variables: Insertar valores con ${variable}
  • Expresiones: Calcular dentro del texto con ${expresión}
  • Multilínea: Escribir texto en varias líneas sin problemas
const nombre = "Ana";
const edad = 20;

// ❌ Forma antigua (concatenación):
console.log("Hola " + nombre + ", tienes " + edad + " años y el año que viene tendrás " + (edad + 1));

// ✅ Forma moderna (template literal):
console.log(`Hola ${nombre}, tienes ${edad} años y el año que viene tendrás ${edad + 1}`);

// Multilínea:
const mensaje = `
  Bienvenido ${nombre},
  Tu edad actual es: ${edad}
  Eres mayor de edad: ${edad >= 18 ? 'Sí' : 'No'}
`;
console.log(mensaje);

Métodos útiles de strings:

const texto = "Hola Mundo";

// Longitud
console.log(texto.length);              // 10

// Mayúsculas/minúsculas
console.log(texto.toUpperCase());       // "HOLA MUNDO"
console.log(texto.toLowerCase());       // "hola mundo"

// Buscar
console.log(texto.includes("Mundo"));   // true
console.log(texto.startsWith("Hola"));  // true
console.log(texto.endsWith("Mundo"));   // true

// Reemplazar
console.log(texto.replace("Mundo", "JavaScript"));  // "Hola JavaScript"

// Dividir
console.log(texto.split(" "));          // ["Hola", "Mundo"]

// Quitar espacios
const conEspacios = "  hola  ";
console.log(conEspacios.trim());        // "hola"

// Acceder a caracteres
console.log(texto[0]);                  // "H"
console.log(texto.charAt(0));           // "H"

3. Boolean (booleanos)

Los booleanos representan valores lógicos: verdadero o falso. Igual que en Java: 

const esMayorDeEdad = true;
const estaLloviendo = false;

// Se usan principalmente en condiciones:
if (esMayorDeEdad) {
  console.log("Puede entrar");
}

// Operaciones lógicas:
const tieneLicencia = true;
const tieneExperiencia = false;

console.log(tieneLicencia && tieneExperiencia);  // false (AND)
console.log(tieneLicencia || tieneExperiencia);  // true (OR)
console.log(!tieneLicencia);                     // false (NOT)

Valores "truthy" y "falsy":

En JavaScript, algunos valores se consideran "falsos" (falsy) cuando se evalúan en un contexto booleano, aunque no sean exactamente false: 

// Valores FALSY (se evalúan como false):
false
0
"" (string vacío)
null
undefined
NaN

// Valores TRUTHY (se evalúan como true):
// ¡TODO lo demás! Incluidos:
true
1, -1, 100 (cualquier número excepto 0)
"hola", "0", " " (cualquier string no vacío)
[], {} (arrays y objetos vacíos)

// Ejemplos:
if (0) {
  console.log("No se ejecuta");  // 0 es falsy
}

if ("hola") {
  console.log("SÍ se ejecuta");  // string no vacío es truthy
}

// Útil para validaciones:
const nombre = "";
if (!nombre) {
  console.log("El nombre está vacío");  // Se ejecuta porque "" es falsy
}

4. Undefined

undefined significa "sin definir". Es el valor que tiene una variable cuando se declara pero no se le asigna ningún valor: 

let nombre;              // Declarada pero sin valor
console.log(nombre);     // undefined
console.log(typeof nombre);  // "undefined"

// También cuando una función no devuelve nada:
function sinReturn() {
  console.log("Hola");
}
const resultado = sinReturn();
console.log(resultado);  // undefined

// O cuando accedes a una propiedad inexistente:
const persona = { nombre: "Ana" };
console.log(persona.edad);  // undefined (no existe la propiedad)

5. Null

null representa la ausencia intencional de valor. Es como decir "esta variable no tiene ningún valor a propósito": 

let usuario = null;      // Intencionalmente sin valor
console.log(usuario);    // null

// Diferencia con undefined:
let a;                   // undefined (no se asignó nada)
let b = null;            // null (se asignó intencionalmente "nada")

// null se usa cuando quieres indicar explícitamente que no hay valor:
let resultadoBusqueda = null;  // Todavía no se buscó
// ...después de buscar...
resultadoBusqueda = { id: 1, nombre: "Ana" };

Diferencia clave:

  • undefined → "No tiene valor porque nunca se le dio uno"
  • null → "No tiene valor porque explícitamente se decidió que no lo tenga"

6. Symbol (avanzado)

Los símbolos son identificadores únicos. No los usaremos mucho al principio, pero es bueno saber que existen: 

const simbolo1 = Symbol("descripción");
const simbolo2 = Symbol("descripción");
console.log(simbolo1 === simbolo2);  // false (cada Symbol es único)

7. BigInt (números muy grandes)

Para números enteros extremadamente grandes que no caben en number: 

const numeroGrande = 9007199254740991n;  // Nota la 'n' al final
const otroGrande = BigInt("9007199254740991");

Comprobar el tipo: typeof

El operador typeof te dice qué tipo de dato es algo: 

console.log(typeof 42);           // "number"
console.log(typeof "hola");       // "string"
console.log(typeof true);         // "boolean"
console.log(typeof undefined);    // "undefined"
console.log(typeof null);         // "object" ⚠️ (bug histórico de JS)
console.log(typeof Symbol());     // "symbol"
console.log(typeof 100n);         // "bigint"
console.log(typeof [1,2,3]);      // "object" (arrays son objetos)
console.log(typeof {a:1});        // "object"
console.log(typeof function(){}); // "function"

Conversión entre tipos

JavaScript puede convertir automáticamente entre tipos (coerción), pero es mejor hacerlo explícitamente para evitar errores: 

// String → Number
const textoNumero = "42";
const numero = Number(textoNumero);
console.log(numero + 8);              // 50

// Alternativas:
console.log(parseInt("42"));          // 42 (entero)
console.log(parseFloat("3.14"));      // 3.14 (decimal)
console.log(+"42");                   // 42 (conversión rápida con +)

// Number → String
const edad = 20;
const edadTexto = String(edad);
console.log(edadTexto + "8");         // "208" (concatenación)
// O más simple:
console.log(edad + "");               // "20"
console.log(edad.toString());         // "20"

// Cualquier cosa → Boolean
console.log(Boolean(1));              // true
console.log(Boolean(0));              // false
console.log(Boolean(""));             // false
console.log(Boolean("hola"));         // true

// Conversión automática (coerción) - ⚠️ Cuidado:
console.log("5" + 3);                 // "53" (concatenación)
console.log("5" - 3);                 // 2 (resta)
console.log("5" * 3);                 // 15 (multiplicación)
console.log("hola" - 3);              // NaN

⚠️ Consejo: Evita confiar en la conversión automática de tipos. Es mejor convertir explícitamente con Number(), String(), etc. para que tu código sea más claro y predecible.

Operadores en JavaScript

Los operadores son símbolos que realizan operaciones sobre valores y variables. Ya conoces muchos de Java: +, -, *, /... En JavaScript son muy similares, pero con algunas diferencias importantes.

Operadores aritméticos

Para realizar operaciones matemáticas:

let a = 10;
let b = 3;

// Operaciones básicas (igual que en Java)
console.log(a + b);    // 13 (suma)
console.log(a - b);    // 7  (resta)
console.log(a * b);    // 30 (multiplicación)
console.log(a / b);    // 3.333... (división)

// ⚠️ DIFERENCIA CON JAVA:
// En Java: 10 / 3 = 3 (división entera entre int)
// En JavaScript: 10 / 3 = 3.333... (SIEMPRE devuelve decimal)

console.log(a % b);    // 1 (módulo: resto de la división)
console.log(a ** b);   // 1000 (potencia: 10³)

El operador módulo (%) es muy útil:

// Comprobar si un número es par:
const numero = 8;
if (numero % 2 === 0) {
  console.log("Es par");
}

// Obtener el último dígito:
console.log(12345 % 10);  // 5

// Hacer que algo se repita cíclicamente:
const dias = ["Lun", "Mar", "Mié", "Jue", "Vie", "Sáb", "Dom"];
const dia = 15;
console.log(dias[dia % 7]);  // "Mar" (día 15 % 7 = 1)

Operadores de asignación

Para asignar y modificar valores:

let x = 10;

// Asignación simple
x = 5;            // x ahora es 5

// Asignación con operación (igual que en Java)
x += 3;           // x = x + 3  →  x = 8
x -= 2;           // x = x - 2  →  x = 6
x *= 2;           // x = x * 2  →  x = 12
x /= 3;           // x = x / 3  →  x = 4
x %= 3;           // x = x % 3  →  x = 1

// Incremento y decremento
x++;              // x = x + 1  →  x = 2
x--;              // x = x - 1  →  x = 1

// Pre-incremento vs post-incremento (igual que en Java)
let y = 5;
console.log(y++);  // Muestra 5, luego incrementa (y ahora es 6)
console.log(++y);  // Incrementa primero, luego muestra 7

Operadores de comparación

Para comparar valores. ⚠️ MUY IMPORTANTE: En JavaScript hay dos tipos de comparación de igualdad, y es crucial entender la diferencia.

Igualdad laxa (==) vs Igualdad estricta (===)

// == : Igualdad LAXA (convierte tipos)
console.log(5 == "5");       // true ❌ (convierte "5" a número)
console.log(1 == true);      // true ❌ (convierte true a 1)
console.log(0 == false);     // true ❌ (convierte false a 0)
console.log(null == undefined);  // true ❌

// === : Igualdad ESTRICTA (compara valor Y tipo)
console.log(5 === "5");      // false ✅ (number !== string)
console.log(1 === true);     // false ✅ (number !== boolean)
console.log(0 === false);    // false ✅ (number !== boolean)
console.log(null === undefined); // false ✅ (null !== undefined)

// Ejemplos correctos:
console.log(5 === 5);        // true
console.log("hola" === "hola"); // true
console.log(true === true);  // true

⚠️ REGLA DE ORO: SIEMPRE usa === y !== (estrictos). NUNCA uses == y != (laxos) a menos que tengas una razón muy específica y sepas exactamente lo que haces.

La igualdad laxa (==) ha causado innumerables bugs a lo largo de la historia de JavaScript. Usa === para evitar sorpresas.

Otros operadores de comparación

const a = 10;
const b = 5;

console.log(a > b);      // true  (mayor que)
console.log(a < b);      // false (menor que)
console.log(a >= 10);    // true  (mayor o igual)
console.log(a <= 5);     // false (menor o igual)
console.log(a !== b);    // true  (diferente - estricto)

Operadores lógicos

Para combinar condiciones booleanas (igual que en Java):

// AND (&&) - Todas las condiciones deben ser verdaderas
const edad = 20;
const tieneLicencia = true;

if (edad >= 18 && tieneLicencia) {
  console.log("Puede conducir");  // Se ejecuta
}

// OR (||) - Al menos una condición debe ser verdadera
const esFinDeSemana = false;
const esFestivo = true;

if (esFinDeSemana || esFestivo) {
  console.log("No hay clase");  // Se ejecuta
}

// NOT (!) - Invierte el valor booleano
const llueve = false;
if (!llueve) {
  console.log("Puedes salir sin paraguas");  // Se ejecuta
}

// Combinaciones:
const temperatura = 25;
const estaLloviendo = false;

if (temperatura > 20 && !estaLloviendo) {
  console.log("Buen día para salir");
}

Cortocircuito (short-circuit)

Los operadores && y || evalúan de izquierda a derecha y se detienen en cuanto saben el resultado: 

// AND (&&) - Si el primero es false, ya no evalúa el resto
false && console.log("No se muestra");  // No ejecuta el console.log

// OR (||) - Si el primero es true, ya no evalúa el resto
true || console.log("No se muestra");   // No ejecuta el console.log

// Uso práctico: valores por defecto
const nombre = null;
const nombreFinal = nombre || "Invitado";
console.log(nombreFinal);  // "Invitado" (porque nombre es null/falsy)

// Ejemplo real:
function saludar(nombre) {
  const nombreAUsar = nombre || "Usuario";  // Si no hay nombre, usa "Usuario"
  console.log(`Hola ${nombreAUsar}`);
}

saludar("Ana");    // "Hola Ana"
saludar();         // "Hola Usuario"

Operador ternario (condicional)

Una forma compacta de escribir un if...else simple: 

// Sintaxis: condición ? valorSiTrue : valorSiFalse

const edad = 20;
const mensaje = (edad >= 18) ? "Mayor de edad" : "Menor de edad";
console.log(mensaje);  // "Mayor de edad"

// Es equivalente a:
let mensaje;
if (edad >= 18) {
  mensaje = "Mayor de edad";
} else {
  mensaje = "Menor de edad";
}

// Más ejemplos:
const nota = 7;
const resultado = (nota >= 5) ? "Aprobado" : "Suspenso";

const precio = 100;
const precioFinal = (precio > 50) ? precio * 0.9 : precio;  // 10% descuento si > 50

// ⚠️ No abuses: si se vuelve complejo, usa if...else normal
const tipo = (x > 0) ? "positivo" : (x < 0) ? "negativo" : "cero";  // Difícil de leer

Operador typeof

Para saber el tipo de un valor:

console.log(typeof 42);           // "number"
console.log(typeof "hola");       // "string"
console.log(typeof true);         // "boolean"
console.log(typeof undefined);    // "undefined"
console.log(typeof {a: 1});       // "object"
console.log(typeof [1, 2, 3]);    // "object" (arrays son objetos)
console.log(typeof null);         // "object" (bug histórico de JS)

Operador de concatenación de strings

El operador + con strings realiza concatenación (igual que en Java): 

const nombre = "Ana";
const apellido = "García";

console.log(nombre + " " + apellido);  // "Ana García"

// ⚠️ Cuidado con la coerción:
console.log("5" + 3);      // "53" (concatenación, no suma)
console.log(5 + 3);        // 8 (suma)
console.log("5" + 3 + 2);  // "532" (todo se concatena)
console.log(5 + 3 + "2");  // "82" (primero suma 5+3, luego concatena)

// Mejor usar template literals:
console.log(`${nombre} ${apellido}`);  // "Ana García" (más claro)

  • ✅ Usa === y !== (SIEMPRE estrictos)
  • ✅ && para "Y lógico", || para "O lógico"
  • ✅ % para obtener el resto de una división
  • ✅ ** para potencias
  • ✅ Template literals en lugar de concatenación con +

Estructuras condicionales

Las estructuras condicionales permiten que nuestro programa tome decisiones y ejecute diferentes bloques de código según se cumplan o no ciertas condiciones. Sin ellas, nuestros programas serían lineales y aburridos, siempre harían exactamente lo mismo.

Imagina un programa que comprueba si un usuario puede entrar a un sitio web. Necesitamos preguntar: "¿Es mayor de 18 años?". Si la respuesta es sí, le dejamos entrar; si es no, le mostramos un mensaje de error. Esto es una estructura condicional.

if: La condición básica

La estructura más simple es el if (si). Funciona exactamente igual que en Java: 

const edad = 20;

if (edad >= 18) {
  console.log("Eres mayor de edad");
  console.log("Puedes entrar");
}

console.log("Este mensaje siempre se muestra");

¿Cómo funciona?

  1. Se evalúa la condición entre paréntesis: edad >= 18
  2. Si es true, se ejecuta todo el bloque entre llaves {...}
  3. Si es false, se salta el bloque y continúa después

if...else: Dos caminos posibles

Cuando queremos hacer algo si la condición es verdadera, y otra cosa diferente si es falsa, usamos else (si no): 

const edad = 16;

if (edad >= 18) {
  console.log("Eres mayor de edad");
  console.log("Puedes acceder al contenido");
} else {
  console.log("Eres menor de edad");
  console.log("No puedes acceder");
}

// Siempre se ejecuta UNO de los dos bloques, nunca ambos

Ejemplo práctico: Descuento en tienda

const precioOriginal = 100;
const esCliente = true;
let precioFinal;

if (esCliente) {
  precioFinal = precioOriginal * 0.9;  // 10% descuento
  console.log("Descuento aplicado");
} else {
  precioFinal = precioOriginal;
  console.log("Sin descuento");
}

console.log(`Precio final: ${precioFinal}€`);

if...else if...else: Múltiples condiciones

Cuando tenemos más de dos posibilidades, encadenamos condiciones con else if: 

const nota = 7.5;

if (nota >= 9) {
  console.log("Sobresaliente");
} else if (nota >= 7) {
  console.log("Notable");
} else if (nota >= 6) {
  console.log("Bien");
} else if (nota >= 5) {
  console.log("Aprobado");
} else {
  console.log("Suspenso");
}

⚠️ ¡IMPORTANTE! Orden de evaluación:

Las condiciones se evalúan de arriba a abajo, y se ejecuta el primer bloque cuya condición sea verdadera. El resto de condiciones YA NO se evalúan: 

const nota = 8;

if (nota >= 5) {
  console.log("Aprobado");     // ✅ Se ejecuta esto
} else if (nota >= 7) {
  console.log("Notable");      // ❌ Nunca llega aquí, aunque nota >= 7
}

// ❌ MAL: Orden incorrecto
// Siempre mostrará "Aprobado" incluso con un 9

// ✅ BIEN: Orden de mayor a menor
if (nota >= 9) {
  console.log("Sobresaliente");
} else if (nota >= 7) {
  console.log("Notable");
} else if (nota >= 5) {
  console.log("Aprobado");
} else {
  console.log("Suspenso");
}

Regla práctica: Cuando uses rangos numéricos, ordena las condiciones de mayor a menor (o de menor a mayor, pero consistentemente).

switch: Para valores exactos

Cuando necesitas comparar una variable contra múltiples valores exactos, switch es más claro que muchos if...else if: 

const dia = 3;

switch (dia) {
  case 1:
    console.log("Lunes");
    break;
  case 2:
    console.log("Martes");
    break;
  case 3:
    console.log("Miércoles");  // ✅ Se ejecuta este
    break;
  case 4:
    console.log("Jueves");
    break;
  case 5:
    console.log("Viernes");
    break;
  case 6:
  case 7:
    console.log("Fin de semana");  // Para 6 o 7
    break;
  default:
    console.log("Día no válido");
}

⚠️ No olvides el break!

El break es fundamental. Sin él, la ejecución "cae" al siguiente caso (fall-through): 

const opcion = "b";

switch (opcion) {
  case "a":
    console.log("Opción A");
    // ❌ Falta break
  case "b":
    console.log("Opción B");  // Se ejecuta
    // ❌ Falta break
  case "c":
    console.log("Opción C");  // También se ejecuta (fall-through)
    break;
}

// Salida:
// "Opción B"
// "Opción C"  ← No queríamos esto

// ✅ CORRECTO: con break
switch (opcion) {
  case "a":
    console.log("Opción A");
    break;  // ✅
  case "b":
    console.log("Opción B");
    break;  // ✅
  case "c":
    console.log("Opción C");
    break;  // ✅
}

Cuándo usar switch vs if:

Usa if...else if Usa switch
Rangos numéricos (if (edad >= 18)) Valores exactos (case 1:, case "a":)
Condiciones complejas (if (a > 5 && b < 10)) Comparar una variable contra muchos valores
Comparaciones con >, <, etc. Menús de opciones, días de la semana, estados...

Operador ternario: if...else en una línea

Para asignaciones condicionales simples, existe una sintaxis compacta: 

// Sintaxis: condición ? valorSiTrue : valorSiFalse

const edad = 20;
const mensaje = (edad >= 18) ? "Mayor" : "Menor";
console.log(mensaje);  // "Mayor"

// Es exactamente igual a:
let mensaje;
if (edad >= 18) {
  mensaje = "Mayor";
} else {
  mensaje = "Menor";
}

// Más ejemplos útiles:
const precio = 100;
const tieneDescuento = true;
const precioFinal = tieneDescuento ? precio * 0.9 : precio;

const numero = 7;
const tipo = (numero % 2 === 0) ? "par" : "impar";

// En expresiones:
console.log(`El número ${numero} es ${numero % 2 === 0 ? "par" : "impar"}`);

💡 Cuándo usar el operador ternario:

  • ✅ Asignaciones simples: const x = condicion ? a : b;
  • ✅ Valores en expresiones: console.log(edad >= 18 ? "Sí" : "No")
  • ❌ NO para lógica compleja (usa if...else normal)
  • ❌ NO anides muchos ternarios (se vuelve ilegible)

Valores truthy y falsy

Una característica peculiar de JavaScript es que cualquier valor puede usarse en una condición, no solo booleanos: 

// Valores FALSY (se evalúan como false):
if (0) { }              // No se ejecuta
if ("") { }             // No se ejecuta (string vacío)
if (null) { }           // No se ejecuta
if (undefined) { }      // No se ejecuta
if (NaN) { }            // No se ejecuta
if (false) { }          // Obviamente no se ejecuta

// Valores TRUTHY (se evalúan como true):
if (1) { }              // Se ejecuta
if (-1) { }             // Se ejecuta (cualquier número != 0)
if ("hola") { }         // Se ejecuta (cualquier string no vacío)
if (" ") { }            // Se ejecuta (espacio no es vacío)
if ([]) { }             // Se ejecuta (array vacío es truthy)
if ({}) { }             // Se ejecuta (objeto vacío es truthy)
if (true) { }           // Obviamente se ejecuta

Aplicación práctica: validaciones

// Comprobar si un string no está vacío:
const nombre = "";

if (nombre) {
  console.log(`Hola ${nombre}`);
} else {
  console.log("Por favor, introduce tu nombre");  // Se ejecuta
}

// Valores por defecto:
function saludar(nombre) {
  if (!nombre) {  // Si nombre es falsy (vacío, null, undefined...)

}

// O más corto:
function saludar(nombre) {
  nombre = nombre || "Invitado";  // Si nombre es falsy, usa "Invitado"

Bucles

Los bucles (loops) son estructuras que nos permiten repetir un bloque de código múltiples veces. Imagina que necesitas mostrar los números del 1 al 100 en consola. Sin bucles, tendrías que escribir 100 líneas de console.log(). Con un bucle, lo haces en 3 líneas.

Los bucles son fundamentales en programación porque nos permiten procesar listas de datos, repetir operaciones, y automatizar tareas repetitivas.

Bucle while: Mientras se cumpla la condición

El bucle while repite un bloque mientras una condición sea verdadera: 

let contador = 1;

while (contador <= 5) {
  console.log("Iteración número " + contador);
  contador++;  // ⚠️ IMPORTANTE: incrementar para evitar bucle infinito
}

console.log("Fin del bucle");

// Salida:
// Iteración número 1
// Iteración número 2
// Iteración número 3
// Iteración número 4
// Iteración número 5
// Fin del bucle

¿Cómo funciona?

  1. Se evalúa la condición contador <= 5
  2. Si es true, se ejecuta el bloque
  3. Se vuelve al paso 1
  4. Cuando la condición es false, sale del bucle

⚠️ ¡Cuidado con los bucles infinitos! Si olvidas modificar la variable que controla la condición (en este caso, contador++), el bucle nunca terminará y tu navegador se colgará: 

let x = 1;
while (x <= 5) {
  console.log(x);
  // ❌ Olvidamos hacer x++
  // Este bucle NUNCA termina

Ejemplo práctico: Suma acumulada

// Sumar los números del 1 al 100
let suma = 0;
let numero = 1;

while (numero <= 100) {
  suma = suma + numero;  // o: suma += numero;
  numero++;
}

console.log("La suma de 1 a 100 es: " + suma);  // 5050

Bucle do...while: Ejecuta al menos una vez

Similar a while, pero la condición se evalúa al FINAL. Esto garantiza que el bloque se ejecute al menos una vez: 

let numero = 10;

do {
  console.log("Número: " + numero);
  numero++;
} while (numero < 10);

// Se ejecuta UNA VEZ aunque la condición sea false desde el principio
// Salida: "Número: 10"

Cuándo usar do...while: Cuando necesites que el código se ejecute al menos una vez, por ejemplo, al pedir datos al usuario:

let respuesta;
do {
  respuesta = prompt("¿Cuál es la capital de España?");
} while (respuesta !== "Madrid");

console.log("¡Correcto!");

Bucle for: La forma más común

El bucle for es el más usado cuando sabes cuántas veces quieres repetir algo: 

// Sintaxis: for (inicialización; condición; actualización)
for (let i = 1; i <= 5; i++) {
  console.log("Iteración " + i);
}

// Salida:
// Iteración 1
// Iteración 2
// Iteración 3
// Iteración 4
// Iteración 5

Partes del bucle for:

for (let i = 0; i < 10; i++) {
     //   │        │        │
     //   │        │        └─ Actualización: se ejecuta al final de cada iteración
     //   │        └─ Condición: mientras sea true, continúa
     //   └─ Inicialización: se ejecuta UNA VEZ al principio

  console.log(i);
}

Ejemplos comunes:

// Contar del 1 al 10
for (let i = 1; i <= 10; i++) {
  console.log(i);
}

// Cuenta atrás del 10 al 1
for (let i = 10; i >= 1; i--) {
  console.log(i);
}

// Solo números pares del 0 al 20
for (let i = 0; i <= 20; i += 2) {
  console.log(i);
}

// O con condicional:
for (let i = 0; i <= 20; i++) {
  if (i % 2 === 0) {
    console.log(i);
  }
}

// Tabla de multiplicar del 5
for (let i = 1; i <= 10; i++) {
  console.log(`5 × ${i} = ${5 * i}`);
}

for...of: Recorrer arrays (IMPORTANTE)

Cuando quieres recorrer un array (lista) completo, for...of es más sencillo y legible: 

const frutas = ["manzana", "pera", "naranja", "plátano"];

// Forma tradicional (funciona, pero verbosa):
for (let i = 0; i < frutas.length; i++) {
  console.log(frutas[i]);
}

// Forma moderna (recomendada):
for (let fruta of frutas) {
  console.log(fruta);
}

// Salida:
// manzana
// pera
// naranja
// plátano

¿Por qué usar for...of?

  • Más legible: for (let item of array)
  • No necesitas índices si no los usas
  • Menos propenso a errores (no hay i < array.length)
  • Funciona con cualquier "iterable" (arrays, strings, Sets, Maps...)
// Recorrer un string (también es iterable):
const texto = "Hola";
for (let letra of texto) {
  console.log(letra);
}
// H
// o
// l
// a

// Si necesitas el índice, usa forEach o for tradicional:
frutas.forEach((fruta, indice) => {
  console.log(`${indice}: ${fruta}`);
});

break y continue: Controlar el flujo

Dentro de un bucle, puedes usar dos palabras clave especiales:

break: Salir del bucle

// Buscar un número en un array
const numeros = [3, 7, 12, 8, 15, 20];
let encontrado = false;

for (let numero of numeros) {
  if (numero > 10) {
    console.log("Encontrado: " + numero);
    encontrado = true;
    break;  // Sale del bucle inmediatamente
  }
}

// Solo muestra: "Encontrado: 12"
// No sigue recorriendo el resto

continue: Saltar a la siguiente iteración

// Mostrar solo números impares del 1 al 10
for (let i = 1; i <= 10; i++) {
  if (i % 2 === 0) {
    continue;  // Salta el resto del código y va a la siguiente iteración
  }
  console.log(i);
}

// Salida: 1, 3, 5, 7, 9

Bucles anidados

Puedes poner un bucle dentro de otro. Útil para tablas, matrices, o combinaciones: 

// Tabla de multiplicar del 1 al 5
for (let i = 1; i <= 5; i++) {
  console.log(`Tabla del ${i}:`);
  for (let j = 1; j <= 10; j++) {
    console.log(`  ${i} × ${j} = ${i * j}`);
  }
  console.log("---");
}

// Generar todas las combinaciones
const colores = ["rojo", "azul", "verde"];
const tamaños = ["S", "M", "L"];

for (let color of colores) {
  for (let tamaño of tamaños) {
    console.log(`${color} - ${tamaño}`);
  }
}

// Salida:
// rojo - S
// rojo - M
// rojo - L
// azul - S
// azul - M
// etc...

⚠️ Cuidado con bucles anidados: Pueden volverse lentos rápidamente. Si tienes 1000 elementos en cada array, el bucle se ejecutará 1.000.000 de veces.

Funciones

Las funciones son uno de los conceptos más importantes en programación. Una función es un bloque de código con un nombre que puedes ejecutar cuantas veces quieras sin tener que reescribir el código.

Piensa en las funciones como recetas: defines una vez cómo hacer algo (la receta), y luego puedes "ejecutarla" (cocinar) todas las veces que quieras. Esto te permite:

  • Reutilizar código: No repetir lo mismo mil veces
  • Organizar: Dividir programas grandes en partes pequeñas
  • Mantener: Si necesitas cambiar algo, lo cambias en un solo sitio
  • Abstraer: Ocultar complejidad detrás de un nombre simple

Declaración básica de funciones

// Declarar una función
function saludar() {
  console.log("¡Hola!");
  console.log("Bienvenido al curso");
}

// Llamar (ejecutar) la función
saludar();  // Se ejecutan las dos líneas de código

// Puedes llamarla cuantas veces quieras:
saludar();
saludar();
saludar();

Anatomía de una función:

function nombreFuncion() {
  //      │              │
  //      │              └─ Paréntesis (aquí irán los parámetros)
  //      └─ Nombre descriptivo en camelCase

  // Cuerpo de la función (código que se ejecuta)
  console.log("Hola");
}

Funciones con parámetros

Los parámetros son valores que le pasas a la función para que trabaje con ellos: 

// Función con UN parámetro
function saludar(nombre) {
  console.log(`¡Hola ${nombre}!`);
}

saludar("Ana");      // "¡Hola Ana!"
saludar("Luis");     // "¡Hola Luis!"
saludar("María");    // "¡Hola María!"

// Función con MÚLTIPLES parámetros
function sumar(a, b) {
  const resultado = a + b;
  console.log(`${a} + ${b} = ${resultado}`);
}

sumar(5, 3);   // "5 + 3 = 8"
sumar(10, 20); // "10 + 20 = 30"

Parámetros vs Argumentos:

  • Parámetros: Variables en la definición de la función (a, b en la declaración)
  • Argumentos: Valores reales que pasas al llamarla (5, 3 en la llamada)

Funciones que devuelven valores (return)

Las funciones pueden devolver un valor usando return. Es como si la función fuera una pregunta y return es la respuesta: 

function sumar(a, b) {
  return a + b;  // Devuelve el resultado
}

// Guardar el resultado en una variable:
const resultado = sumar(5, 3);
console.log(resultado);  // 8

// O usarlo directamente:
console.log(sumar(10, 20));  // 30
console.log(sumar(5, 3) * 2);  // 16 (primero suma, luego multiplica)

⚠️ IMPORTANTE sobre return:

  • return termina la función inmediatamente
  • Cualquier código después de return NO se ejecuta
  • Si no hay return, la función devuelve undefined
function ejemplo() {
  console.log("Antes del return");
  return "valor";
  console.log("Después del return");  // ❌ NUNCA se ejecuta
}

const resultado = ejemplo();
// Solo muestra: "Antes del return"
console.log(resultado);  // "valor"

// Función sin return:
function sinReturn() {
  console.log("Hola");
}

const res = sinReturn();
console.log(res);  // undefined

Arrow Functions (funciones flecha)

ES6 (2015) introdujo una sintaxis más compacta para funciones: las arrow functions (funciones flecha). Son especialmente útiles para funciones cortas: 

// Función tradicional:
function sumar(a, b) {
  return a + b;
}

// Arrow function equivalente:
const sumar = (a, b) => {
  return a + b;
};

// Arrow function aún más corta (si solo hay un return):
const sumar = (a, b) => a + b;

// Las tres son equivalentes:
console.log(sumar(5, 3));  // 8

Sintaxis de arrow functions:

// Con múltiples parámetros:
const sumar = (a, b) => a + b;

// Con UN parámetro (se pueden omitir paréntesis):
const cuadrado = n => n * n;
const saludar = nombre => `Hola ${nombre}`;

// Sin parámetros:
const holaM undo = () => console.log("Hola");

// Con cuerpo de múltiples líneas (necesitas llaves y return):
const calcular = (a, b) => {
  const suma = a + b;
  const producto = a * b;
  return { suma, producto };  // Devuelve un objeto
};

console.log(calcular(3, 4));  // { suma: 7, producto: 12 }

¿Cuándo usar arrow functions vs funciones tradicionales?

  • ✅ Arrow functions: Para funciones cortas, callbacks, métodos de arrays
  • ✅ Tradicionales: Cuando necesitas this, métodos de objetos, constructores

Al principio, puedes usar arrow functions para casi todo y funcionará bien.

Parámetros por defecto

Puedes dar valores por defecto a los parámetros por si no se pasan: 

function saludar(nombre = "Invitado") {
  console.log(`Hola ${nombre}`);
}

saludar("Ana");    // "Hola Ana"
saludar();         // "Hola Invitado" (usa el valor por defecto)

// Con múltiples parámetros:
function crearUsuario(nombre, edad = 18, ciudad = "Madrid") {
  return { nombre, edad, ciudad };
}

console.log(crearUsuario("Ana"));
// { nombre: "Ana", edad: 18, ciudad: "Madrid" }

console.log(crearUsuario("Luis", 25, "Barcelona"));
// { nombre: "Luis", edad: 25, ciudad: "Barcelona" }

Ámbito (scope) de variables

Las variables declaradas dentro de una función solo existen dentro de ella (ámbito local). Las variables fuera de todas las funciones son globales: 

// Variable GLOBAL
const nombre = "Ana";

function mostrarDatos() {
  // Variable LOCAL
  const edad = 20;

  console.log(nombre);  // ✅ Accede a variable global
  console.log(edad);    // ✅ Accede a variable local
}

mostrarDatos();

console.log(nombre);  // ✅ Variable global accesible
console.log(edad);    // ❌ ERROR: edad no existe aquí

// Ejemplo con parámetros:
function saludar(persona) {
  // 'persona' es local a esta función
  console.log(`Hola ${persona}`);
}

saludar("Luis");
console.log(persona);  // ❌ ERROR: persona no existe fuera de la función

💡 Buena práctica: Usa variables locales siempre que puedas. Las variables globales pueden causar conflictos en programas grandes.

Funciones como valores

En JavaScript,con las funciones puedes:

  • Asignarlas a variables
  • Pasarlas como argumentos a otras funciones
  • Devolverlas desde otras funciones
// Asignar función a variable:
const duplicar = function(n) {
  return n * 2;
};

console.log(duplicar(5));  // 10

// Pasar función como argumento (callback):
function procesar(numero, operacion) {
  return operacion(numero);
}

const resultado = procesar(5, duplicar);
console.log(resultado);  // 10

// O directamente:
const resultado2 = procesar(5, function(n) {
  return n * 3;
});
console.log(resultado2);  // 15

// Con arrow function:
const resultado3 = procesar(5, n => n * n);
console.log(resultado3);  // 25

Arrays

Un array (arreglo o lista) es una estructura de datos que permite almacenar múltiples valores en una sola variable.

En Java, los arrays son de tamaño fijo y tipo fijo (int[] numeros = new int[5];). En JavaScript, los arrays son más flexibles: pueden crecer o reducirse dinámicamente, y pueden contener elementos de diferentes tipos.

Crear arrays

// Array vacío
const lista = [];

// Array con elementos
const numeros = [1, 2, 3, 4, 5];
const frutas = ["manzana", "pera", "naranja"];

// Array con tipos mixtos (posible en JS, pero no recomendado):
const mixto = [42, "hola", true, null];

// Array de objetos (muy común):
const estudiantes = [
  { nombre: "Ana", nota: 8 },
  { nombre: "Luis", nota: 7 },
  { nombre: "María", nota: 9 }
];

Acceder a elementos

Los arrays en JavaScript (como en Java) usan índices que empiezan en 0: 

const frutas = ["manzana", "pera", "naranja", "plátano"];
//              índice:  0         1        2         3

console.log(frutas[0]);  // "manzana" (primer elemento)
console.log(frutas[2]);  // "naranja" (tercer elemento)
console.log(frutas[3]);  // "plátano" (último elemento)

// Acceder al último elemento (cuando no sabes la longitud):
console.log(frutas[frutas.length - 1]);  // "plátano"

// Intentar acceder a un índice que no existe:
console.log(frutas[10]);  // undefined (no da error, solo undefined)

Modificar elementos

const numeros = [10, 20, 30];

// Modificar un elemento existente:
numeros[1] = 99;
console.log(numeros);  // [10, 99, 30]

// ⚠️ Aunque el array sea const, SÍ puedes modificar sus elementos
// Lo que NO puedes es reasignar el array completo:
const lista = [1, 2, 3];
lista[0] = 99;        // ✅ Permitido
lista = [4, 5, 6];    // ❌ ERROR: no puedes reasignar

Propiedades y métodos básicos

length: Longitud del array

const frutas = ["manzana", "pera", "naranja"];

console.log(frutas.length);  // 3

// Útil en bucles:
for (let i = 0; i < frutas.length; i++) {
  console.log(frutas[i]);
}

push(): Añadir al final

const numeros = [1, 2, 3];

numeros.push(4);
console.log(numeros);  // [1, 2, 3, 4]

numeros.push(5, 6);    // Puedes añadir varios
console.log(numeros);  // [1, 2, 3, 4, 5, 6]

pop(): Eliminar del final

const numeros = [1, 2, 3, 4];

const ultimo = numeros.pop();  // Devuelve el elemento eliminado
console.log(ultimo);   // 4
console.log(numeros);  // [1, 2, 3]

unshift(): Añadir al principio

const numeros = [2, 3, 4];

numeros.unshift(1);
console.log(numeros);  // [1, 2, 3, 4]

shift(): Eliminar del principio

const numeros = [1, 2, 3, 4];

const primero = numeros.shift();
console.log(primero);  // 1
console.log(numeros);  // [2, 3, 4]

Métodos funcionales (MUY IMPORTANTES)

Los métodos funcionales son la forma moderna y elegante de trabajar con arrays en JavaScript. Son especialmente útiles porque:

  • No modifican el array original (son inmutables)
  • Son más legibles y expresivos
  • Permiten encadenar operaciones
  • Son el estándar en programación funcional

forEach(): Recorrer el array

forEach ejecuta una función para cada elemento. No devuelve nada (o mejor dicho, devuelve undefined): 

const numeros = [1, 2, 3, 4, 5];

// Mostrar cada número:
numeros.forEach(function(numero) {
  console.log(numero);
});

// Con arrow function (más común):
numeros.forEach(numero => {
  console.log(numero);
});

// Con índice (si lo necesitas):
numeros.forEach((numero, indice) => {
  console.log(`Posición ${indice}: ${numero}`);
});

// Salida:
// Posición 0: 1
// Posición 1: 2
// etc...

map(): Transformar cada elemento

map crea un nuevo array aplicando una función a cada elemento. No modifica el original: 

const numeros = [1, 2, 3, 4, 5];

// Duplicar cada número:
const dobles = numeros.map(n => n * 2);

console.log(numeros);  // [1, 2, 3, 4, 5] (original sin cambios)
console.log(dobles);   // [2, 4, 6, 8, 10] (nuevo array)

// Ejemplos más complejos:
const nombres = ["ana", "luis", "maría"];

// Convertir a mayúsculas:
const nombresMayus = nombres.map(nombre => nombre.toUpperCase());
console.log(nombresMayus);  // ["ANA", "LUIS", "MARÍA"]

// Con objetos:
const estudiantes = [
  { nombre: "Ana", nota: 8 },
  { nombre: "Luis", nota: 7 }
];

// Extraer solo los nombres:
const soloNombres = estudiantes.map(est => est.nombre);
console.log(soloNombres);  // ["Ana", "Luis"]

// Transformar objetos:
const con10Mas = estudiantes.map(est => ({
  nombre: est.nombre,
  nota: est.nota + 1
}));
console.log(con10Mas);
// [{ nombre: "Ana", nota: 9 }, { nombre: "Luis", nota: 8 }]

filter(): Filtrar elementos

filter crea un nuevo array con solo los elementos que cumplen una condición: 

const numeros = [1, 5, 10, 15, 20, 25, 30];

// Solo números mayores que 10:
const mayores = numeros.filter(n => n > 10);
console.log(mayores);  // [15, 20, 25, 30]

// Solo números pares:
const pares = numeros.filter(n => n % 2 === 0);
console.log(pares);  // [10, 20, 30]

// Con objetos:
const estudiantes = [
  { nombre: "Ana", nota: 8 },
  { nombre: "Luis", nota: 4 },
  { nombre: "María", nota: 9 },
  { nombre: "Carlos", nota: 3 }
];

// Solo aprobados (nota >= 5):
const aprobados = estudiantes.filter(est => est.nota >= 5);
console.log(aprobados);
// [{ nombre: "Ana", nota: 8 }, { nombre: "María", nota: 9 }]

// Solo nombres que empiezan con "M":
const conM = estudiantes.filter(est => est.nombre.startsWith("M"));
console.log(conM);  // [{ nombre: "María", nota: 9 }]

find(): Encontrar un elemento

find devuelve el primer elemento que cumple la condición (o undefined si no encuentra ninguno): 

const numeros = [1, 5, 10, 15, 20];

// Encontrar el primer número mayor que 10:
const encontrado = numeros.find(n => n > 10);
console.log(encontrado);  // 15 (el primero que cumple)

// Si no encuentra nada:
const noExiste = numeros.find(n => n > 100);
console.log(noExiste);  // undefined

// Con objetos:
const usuarios = [
  { id: 1, nombre: "Ana" },
  { id: 2, nombre: "Luis" },
  { id: 3, nombre: "María" }
];

const usuario = usuarios.find(u => u.id === 2);
console.log(usuario);  // { id: 2, nombre: "Luis" }

findIndex(): Encontrar la posición

const numeros = [10, 20, 30, 40];

const indice = numeros.findIndex(n => n === 30);
console.log(indice);  // 2

const noExiste = numeros.findIndex(n => n === 100);
console.log(noExiste);  // -1 (no encontrado)

some(): ¿Alguno cumple la condición?

const numeros = [1, 3, 5, 7, 10];

// ¿Hay algún número par?
const hayPares = numeros.some(n => n % 2 === 0);
console.log(hayPares);  // true (el 10 es par)

const estudiantes = [
  { nombre: "Ana", nota: 8 },
  { nombre: "Luis", nota: 4 }
];

// ¿Hay algún suspenso?
const haySuspensos = estudiantes.some(est => est.nota < 5);
console.log(haySuspensos);  // true

every(): ¿Todos cumplen la condición?

const numeros = [2, 4, 6, 8];

// ¿Todos son pares?
const todosPares = numeros.every(n => n % 2 === 0);
console.log(todosPares);  // true

const numeros2 = [2, 4, 5, 8];
console.log(numeros2.every(n => n % 2 === 0));  // false (el 5 no es par)

Encadenar métodos

Lo potente de estos métodos es que puedes encadenarlos: 

const estudiantes = [
  { nombre: "Ana", nota: 8 },
  { nombre: "Luis", nota: 4 },
  { nombre: "María", nota: 9 },
  { nombre: "Carlos", nota: 3 },
  { nombre: "Elena", nota: 7 }
];

// Obtener nombres de aprobados en mayúsculas:
const resultado = estudiantes
  .filter(est => est.nota >= 5)        // Solo aprobados
  .map(est => est.nombre.toUpperCase()) // Nombres en mayúsculas
  .sort();                              // Ordenar alfabéticamente

console.log(resultado);  // ["ANA", "ELENA", "MARÍA"]

Otros métodos útiles

const numeros = [3, 1, 4, 1, 5, 9, 2, 6];

// includes(): ¿Contiene un elemento?
console.log(numeros.includes(5));  // true
console.log(numeros.includes(10)); // false

// indexOf(): Posición de un elemento
console.log(numeros.indexOf(4));   // 2
console.log(numeros.indexOf(99));  // -1 (no existe)

// reverse(): Invertir orden (modifica original)
const invertido = numeros.reverse();
console.log(invertido);  // [6, 2, 9, 5, 1, 4, 1, 3]

// sort(): Ordenar (modifica original)
const ordenado = numeros.sort();
console.log(ordenado);  // [1, 1, 2, 3, 4, 5, 6, 9]

// join(): Convertir a string
const texto = numeros.join(", ");
console.log(texto);  // "1, 1, 2, 3, 4, 5, 6, 9"

// slice(): Extraer porción (NO modifica original)
const porcion = numeros.slice(2, 5);
console.log(porcion);  // [2, 3, 4]
console.log(numeros);  // Original sin cambios

💡 Diferencias clave:

  • forEach: Recorre, no devuelve nada
  • map: Transforma, devuelve nuevo array
  • filter: Filtra, devuelve nuevo array
  • find: Busca, devuelve EL elemento (o undefined)
  • some: Pregunta "¿alguno?", devuelve boolean
  • every: Pregunta "¿todos?", devuelve boolean

Objetos

Los objetos son estructuras de datos que almacenan información en pares clave-valor. Mientras que los arrays almacenan listas ordenadas por índices numéricos, los objetos almacenan datos con nombres descriptivos (propiedades).

En Java, los objetos son instancias de clases. En JavaScript, los objetos son más simples y flexibles: son simplemente colecciones de propiedades que puedes crear directamente sin necesidad de una clase.

Crear objetos literales

// Objeto vacío
const obj = {};

// Objeto con propiedades
const persona = {
  nombre: "Ana",
  edad: 20,
  estudiante: true,
  ciudad: "Valencia"
};

// Con diferentes tipos de valores:
const usuario = {
  id: 1,
  nombre: "Luis",
  activo: true,
  hobbies: ["lectura", "música", "deporte"],  // Array
  direccion: {                                 // Objeto anidado
    calle: "Gran Vía",
    numero: 123,
    ciudad: "Madrid"
  },
  saludar: function() {                        // Función (método)
    console.log("Hola, soy " + this.nombre);
  }
};

Acceder a propiedades

Hay dos formas de acceder a las propiedades de un objeto: 

const persona = {
  nombre: "Ana",
  edad: 20,
  ciudad: "Valencia"
};

// 1. Notación de punto (más común):
console.log(persona.nombre);   // "Ana"
console.log(persona.edad);     // 20

// 2. Notación de corchetes:
console.log(persona["nombre"]); // "Ana"
console.log(persona["edad"]);   // 20

// La notación de corchetes es útil cuando:
// - El nombre de la propiedad está en una variable:
const propiedad = "ciudad";
console.log(persona[propiedad]);  // "Valencia"

// - El nombre tiene espacios o caracteres especiales:
const config = {
  "color principal": "azul",
  "tamaño-fuente": "16px"
};
console.log(config["color principal"]);  // "azul"

// Acceder a propiedades inexistentes devuelve undefined:
console.log(persona.apellido);  // undefined

Modificar y añadir propiedades

const persona = {
  nombre: "Ana",
  edad: 20
};

// Modificar propiedad existente:
persona.edad = 21;
console.log(persona.edad);  // 21

// Añadir nueva propiedad:
persona.ciudad = "Madrid";
persona.email = "ana@mail.com";

console.log(persona);
// { nombre: "Ana", edad: 21, ciudad: "Madrid", email: "ana@mail.com" }

// Eliminar propiedad:
delete persona.email;
console.log(persona);
// { nombre: "Ana", edad: 21, ciudad: "Madrid" }

Métodos en objetos

Las funciones dentro de objetos se llaman métodos: 

const persona = {
  nombre: "Ana",
  edad: 20,

  // Método (función dentro del objeto):
  saludar: function() {
    console.log(`Hola, soy ${this.nombre}`);
  },

  // Sintaxis corta (ES6):
  despedir() {
    console.log(`Adiós, soy ${this.nombre}`);
  },

  cumplirAños() {
    this.edad++;
    console.log(`Ahora tengo ${this.edad} años`);
  }
};

persona.saludar();      // "Hola, soy Ana"
persona.despedir();     // "Adiós, soy Ana"
persona.cumplirAños();  // "Ahora tengo 21 años"

⚠️ La palabra clave this:

Dentro de un método, this se refiere al objeto mismo. Es como decir "yo" o "este objeto": 

const calculadora = {
  resultado: 0,

  sumar(n) {
    this.resultado += n;  // this.resultado es "mi propiedad resultado"
    return this;          // Devolver this permite encadenar
  },

  restar(n) {
    this.resultado -= n;
    return this;
  },

  mostrar() {
    console.log("Resultado: " + this.resultado);
    return this;
  }
};

// Encadenar métodos:
calculadora.sumar(10).sumar(5).restar(3).mostrar();
// "Resultado: 12"

Recorrer objetos

for...in: Recorrer propiedades

const persona = {
  nombre: "Ana",
  edad: 20,
  ciudad: "Valencia"
};

// for...in recorre las CLAVES (nombres de propiedades):
for (let clave in persona) {
  console.log(clave + ": " + persona[clave]);
}

// Salida:
// nombre: Ana
// edad: 20
// ciudad: Valencia

Object.keys(), Object.values(), Object.entries()

const persona = {
  nombre: "Ana",
  edad: 20,
  ciudad: "Valencia"
};

// Obtener array de claves:
const claves = Object.keys(persona);
console.log(claves);  // ["nombre", "edad", "ciudad"]

// Obtener array de valores:
const valores = Object.values(persona);
console.log(valores);  // ["Ana", 20, "Valencia"]

// Obtener array de pares [clave, valor]:
const entradas = Object.entries(persona);
console.log(entradas);
// [["nombre", "Ana"], ["edad", 20], ["ciudad", "Valencia"]]

// Útil para recorrer con forEach:
Object.entries(persona).forEach(([clave, valor]) => {
  console.log(`${clave}: ${valor}`);
});

Arrays de objetos (MUY COMÚN)

Una de las estructuras más comunes en JavaScript es un array de objetos: 

const estudiantes = [
  { nombre: "Ana", edad: 20, nota: 8.5 },
  { nombre: "Luis", edad: 21, nota: 7.0 },
  { nombre: "María", edad: 19, nota: 9.2 },
  { nombre: "Carlos", edad: 22, nota: 6.5 }
];

// Recorrer con forEach:
estudiantes.forEach(est => {
  console.log(`${est.nombre}: ${est.nota}`);
});

// Filtrar aprobados:
const aprobados = estudiantes.filter(est => est.nota >= 5);

// Obtener solo los nombres:
const nombres = estudiantes.map(est => est.nombre);
console.log(nombres);  // ["Ana", "Luis", "María", "Carlos"]

// Buscar un estudiante:
const maria = estudiantes.find(est => est.nombre === "María");
console.log(maria);  // { nombre: "María", edad: 19, nota: 9.2 }

// Calcular nota media:
const sumaNotas = estudiantes.reduce((suma, est) => suma + est.nota, 0);
const media = sumaNotas / estudiantes.length;
console.log("Nota media: " + media);

Desestructuración de objetos

La desestructuración es una sintaxis para extraer propiedades de objetos de forma más concisa: 

const persona = {
  nombre: "Ana",
  edad: 20,
  ciudad: "Valencia"
};

// Sin desestructuración:
const nombre = persona.nombre;
const edad = persona.edad;
const ciudad = persona.ciudad;

// Con desestructuración:
const { nombre, edad, ciudad } = persona;

console.log(nombre);  // "Ana"
console.log(edad);    // 20
console.log(ciudad);  // "Valencia"

// Con nombres diferentes:
const { nombre: n, edad: e } = persona;
console.log(n);  // "Ana"
console.log(e);  // 20

// Con valores por defecto:
const { nombre, pais = "España" } = persona;
console.log(pais);  // "España" (valor por defecto)

// En parámetros de funciones:
function mostrarDatos({ nombre, edad }) {
  console.log(`${nombre} tiene ${edad} años`);
}

mostrarDatos(persona);  // "Ana tiene 20 años"

Copiar objetos

⚠️ IMPORTANTE: Los objetos se copian por referencia, no por valor: 

const original = { nombre: "Ana", edad: 20 };

// ❌ Esto NO hace una copia:
const copia = original;
copia.edad = 25;

console.log(original.edad);  // 25 (¡también cambió!)
// Ambas variables apuntan al MISMO objeto en memoria

// ✅ Para hacer una copia real (superficial):
const copiaReal = { ...original };  // Spread operator
copiaReal.edad = 30;

console.log(original.edad);  // 20 (no cambió)
console.log(copiaReal.edad); // 30

// O con Object.assign():
const otraCopia = Object.assign({}, original);

⚠️ Nota: El spread operator (...) y Object.assign() hacen copias superficiales. Si el objeto tiene objetos anidados, esos siguen siendo referencias.

Ejemplos prácticos

Ejemplo 1: Carrito de compra

const carrito = {
  items: [],

  agregar(producto, cantidad) {
    this.items.push({ producto, cantidad });
    console.log(`Agregado: ${producto} (x${cantidad})`);
  },

  calcularTotal(preciosProductos) {
    let total = 0;
    this.items.forEach(item => {
      total += preciosProductos[item.producto] * item.cantidad;
    });
    return total;
  },

  mostrar() {
    console.log("=== CARRITO ===");
    this.items.forEach(item => {
      console.log(`- ${item.producto}: ${item.cantidad}`);
    });
  }
};

carrito.agregar("manzanas", 3);
carrito.agregar("peras", 2);
carrito.mostrar();

const precios = { manzanas: 2, peras: 3 };
const total = carrito.calcularTotal(precios);
console.log(`Total: ${total}€`);  // Total: 12€

💡 Resumen objetos vs arrays:

  • Arrays: Listas ordenadas por índice (0, 1, 2...). Usa cuando importa el orden.
  • Objetos: Colecciones con claves nombradas. Usa cuando importan los nombres/etiquetas.
  • Arrays de objetos: Muy común para listas de elementos estructurados (usuarios, productos, etc.)

El DOM: Manipular páginas web

Hasta ahora hemos visto JavaScript "puro": variables, funciones, arrays... Pero el verdadero poder de JavaScript en el navegador es poder modificar la página web en tiempo real. Para eso existe el DOM (Document Object Model).

El DOM es una representación del documento HTML en forma de árbol de objetos. Cada elemento HTML (etiquetas como <h1>, <p>, <div>...) se convierte en un objeto que JavaScript puede leer y modificar.

Seleccionar elementos

// Por ID (único):
const titulo = document.getElementById("mi-titulo");

// Por clase (devuelve el primero que encuentre):
const boton = document.querySelector(".btn-primary");

// Por selector CSS (todos los que coincidan):
const parrafos = document.querySelectorAll("p");
const items = document.querySelectorAll(".lista .item");

// querySelector acepta CUALQUIER selector CSS:
const elemento = document.querySelector("#contenedor > div.activo");

Modificar contenido

const titulo = document.getElementById("titulo");

// textContent: Solo texto (seguro)
titulo.textContent = "Nuevo título";

// innerHTML: HTML completo (¡cuidado con seguridad!)
const contenedor = document.getElementById("contenedor");
contenedor.innerHTML = "
Nuevo párrafo
";

// Para inputs:
const nombreInput = document.getElementById("nombre");
const valor = nombreInput.value;  // Leer
nombreInput.value = "Ana García";  // Escribir

Modificar estilos y clases

const caja = document.getElementById("caja");

// Estilos inline:
caja.style.backgroundColor = "blue";
caja.style.color = "white";
caja.style.padding = "20px";

// ✅ MEJOR: Usar clases CSS:
caja.classList.add("activo");     // Añadir clase
caja.classList.remove("oculto");  // Quitar clase
caja.classList.toggle("destacado"); // Alternar clase

if (caja.classList.contains("activo")) {
  console.log("La caja está activa");
}

Crear y eliminar elementos

// Crear nuevo elemento:
const nuevoParrafo = document.createElement("p");
nuevoParrafo.textContent = "Texto nuevo";
nuevoParrafo.classList.add("destacado");

// Añadirlo al DOM:
document.body.appendChild(nuevoParrafo);

// Eliminar elemento:
nuevoParrafo.remove();  // Forma moderna
// o
document.body.removeChild(nuevoParrafo);  // Forma antigua

💡 Pasos para crear elementos:

  1. Crear: const el = document.createElement("tipo");
  2. Configurar: el.textContent = "..."; el.classList.add("...");
  3. Insertar: padre.appendChild(el);

Eventos: Reaccionar a acciones del usuario

Los eventos son acciones que ocurren en el navegador: clics, teclas presionadas, formularios enviados, etc. JavaScript puede "escuchar" estos eventos y ejecutar código cuando ocurren.

addEventListener

const boton = document.getElementById("mi-boton");

boton.addEventListener("click", function() {
  console.log("¡Botón clickeado!");
});

// Con arrow function (más común):
boton.addEventListener("click", () => {
  console.log("¡Botón clickeado!");
});

Tipos de eventos comunes

// RATÓN
elemento.addEventListener("click", () => {});      // Clic
elemento.addEventListener("dblclick", () => {});   // Doble clic
elemento.addEventListener("mouseover", () => {});  // Ratón encima
elemento.addEventListener("mouseout", () => {});   // Ratón sale

// TECLADO
input.addEventListener("keydown", () => {});   // Tecla presionada
input.addEventListener("keyup", () => {});     // Tecla soltada

// FORMULARIO
form.addEventListener("submit", (e) => {
  e.preventDefault();  // ⚠️ IMPORTANTE: evita recargar la página
  // Tu código aquí
});

input.addEventListener("input", () => {});    // Cada vez que cambia el texto
input.addEventListener("change", () => {});   // Al perder el foco
input.addEventListener("focus", () => {});    // Al obtener el foco
input.addEventListener("blur", () => {});     // Al perder el foco

El objeto event

boton.addEventListener("click", function(event) {

  console.log(event.clientY);     // Posición Y del ratón

Formularios y validación

Trabajar con formularios es una de las tareas más comunes en JavaScript. Necesitamos capturar los datos ingresados y validarlos antes de procesarlos.

Capturar el envío del formulario

const form = document.getElementById("mi-form");

form.addEventListener("submit", function(event) {
  event.preventDefault();  // ⚠️ SIEMPRE: evita recargar la página

  // Obtener valores:
  const nombre = document.getElementById("nombre").value;
  const email = document.getElementById("email").value;

  console.log(nombre, email);
});

Validación básica

form.addEventListener("submit", (e) => {
  e.preventDefault();

  const nombre = document.getElementById("nombre").value.trim();
  const email = document.getElementById("email").value.trim();

  // Validar campo vacío:
  if (nombre === "") {
    alert("El nombre es obligatorio");
    return;
  }

  // Validar longitud mínima:
  if (nombre.length < 3) {
    alert("El nombre debe tener al menos 3 caracteres");
    return;
  }

  // Validar formato de email:
  if (!email.includes("@")) {
    alert("Email no válido");
    return;
  }

  // Si todo es válido:
  console.log("Formulario válido:", { nombre, email });
});

⚠️ Nunca olvides:

  • event.preventDefault() en el submit
  • .trim() para quitar espacios
  • Validar ANTES de procesar los datos
